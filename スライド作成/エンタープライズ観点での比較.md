# エンタープライズ観点でのツール比較

「きれいに作れる」だけでは企業導入できない。情報セキュリティ部門が見る観点——データがどこに行くか、誰がアクセスできるか、ログが残るか——で各ツールを比較する。

---

## 結論

| ツール | エンタープライズ適性 | 一言 |
|---|---|---|
| Microsoft 365 Copilot | ◎ | データが Microsoft テナント外に出ない。既存の M365 契約でそのまま使える |
| Claude Enterprise | ○ | SOC 2 Type II、SAML SSO、監査ログ。データは Anthropic インフラ内にとどまる |
| Gamma（Business プラン） | △ | SSO あり、DPA 提供あり、SOC 2 取得中（未完）。成熟度はまだ低い |
| python-pptx + Claude API | ○ | データ経路を自分で制御できる。設計次第で一番安全にできる |

---

## Microsoft 365 Copilot（Copilot in PowerPoint）

企業のほとんどが既に M365 契約を持っているため、**追加ベンダーなしで導入できる**のが最大の強み。

**データ保護の仕組み:**
- プロンプト・レスポンス・生成ファイルはすべて Microsoft 365 テナント内で処理される
- データが LLM のトレーニングに使われない（Microsoft の契約上の保証）
- Microsoft Purview の DLP ポリシーと連携——機密ラベルが付いたファイルを Copilot が処理しないよう制限できる
- EU Data Boundary 対応（欧州データ主権への対応）

**管理機能:**
- M365 管理センター内の「Copilot コントロールシステム」（2025年7月〜）で組織全体の利用ポリシーを一元管理
- Microsoft Purview で全ユーザーの Copilot 操作を監査ログに記録
- 感度ラベルが Copilot 生成コンテンツにも継承される

**コスト:**
- Microsoft 365 Copilot ライセンス（$30/ユーザー/月）が必要
- すでに M365 E3/E5 を契約しているなら追加はこのライセンスだけ

**向いている組織:**
M365 を使っている企業なら最初の選択肢。ISMS・SOC 2 などの認証取得済みの Microsoft インフラに乗っかれるため、情報セキュリティ部門の承認が取りやすい。

---

## Claude Enterprise

Claude の Enterprise プランは AI ツールとしてはかなり成熟したセキュリティ体制を持つ。

**セキュリティ・コンプライアンス:**
- SOC 2 Type II 取得済み
- AES-256 での保存時暗号化
- データはデフォルトでモデルのトレーニングに使われない

**ID・アクセス管理:**
- SAML 2.0 / OIDC によるSSO対応（Okta、Azure AD、Ping Identity と連携済み）
- Just-in-Time プロビジョニング（IdP でユーザーを管理すればアカウントが自動作成される）
- RBAC（ロールベースのアクセス制御）

**管理機能:**
- 組織全体のポリシー設定
- 監査ログ
- Claude in PowerPoint アドインも Enterprise プランで利用可能

**注意点:**
データは Microsoft テナントの外に出て Anthropic のインフラに送られる。M365 の DLP ポリシーとは独立したセキュリティ体制になるため、「すべてのデータを Microsoft のコンプライアンス境界内に収めたい」という要件がある組織には向かない。

**向いている組織:**
Claude 自体をさまざまな業務に使っていて、スライド生成もその一部として統合したい場合。

---

## Gamma（Business / Teams プラン）

**利用できるセキュリティ機能:**
- SSO（Business プラン以上）
- Business プラン以上はデータをモデルトレーニングに使わない
- DPA（データ処理補足契約）を提供

**課題:**
- SOC 2 Type II は取得中であり、2026年3月時点で完了していない
- Microsoft・Anthropic と比較してセキュリティ体制の成熟度が低い
- データが Gamma のインフラ + Gamma が使う AI プロバイダー（複数の LLM）に送られる経路が複雑

**向いている組織:**
個人・小チームでのたたき台作成には最適。しかし ISMS 認証を持つ企業や金融・医療など規制の厳しい業種では情報セキュリティ部門の承認が取りにくい可能性が高い。

---

## python-pptx + Claude API（自前構築）

データ経路を完全に自分でコントロールできる。

**エンタープライズ向けの構成例:**
```
社内データ → 自社 API サーバー（VPC 内）
           → Claude API（または社内展開モデル）
           → python-pptx で PPTX 生成
           → 社内ストレージに保存
```

- 個人情報・機密情報をプロンプトに入れる前にマスキングできる
- Claude API は入力データをトレーニングに使わない
- Amazon Bedrock の Claude モデルを使えば AWS のコンプライアンス体制に乗れる

**向いている組織:**
定型レポートの自動化など特定ユースケースで使いたい開発チームがいる組織。汎用スライドツールとして全社展開するのは難しいが、特定業務の自動化には適している。

## 判断フロー

```
Q1. すでに Microsoft 365 を使っているか？
  → YES: M365 Copilot が最短・最安・最も承認を取りやすい
  → NO: 次へ

Q2. Claude をすでに契約しているか / 他業務でも使う予定があるか？
  → YES: Claude Enterprise でスライド生成も統合
  → NO: 次へ

Q3. 定型レポートの自動化など特定用途か？
  → YES: python-pptx + Claude API（Amazon Bedrock 経由が Enterprise 向けに使いやすい）
  → NO: Gamma は手軽だが社内セキュリティ審査を確認してから
```

## よくある社内承認のポイント

| 審査項目 | M365 Copilot | Claude Enterprise | Gamma |
|---|---|---|---|
| データが学習に使われないか | ✓ 保証あり | ✓ 保証あり | ✓（Business以上） |
| SOC 2 Type II | ✓ | ✓ | △ 取得中 |
| SSO 対応 | ✓ Azure AD 統合 | ✓ SAML/OIDC | ✓（Business以上） |
| 監査ログ | ✓ Microsoft Purview | ✓ | 限定的 |
| DPA 提供 | ✓ | ✓ | ✓ |
| データの保管場所 | Microsoft テナント内 | Anthropic インフラ | Gamma インフラ |
