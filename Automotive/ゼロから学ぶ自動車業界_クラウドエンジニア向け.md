# ゼロから学ぶ自動車業界：クラウドエンジニア向け教材ガイド

## こういう場面のための記事

クラウドエンジニアとして自動車業界の案件に関わるとき、クルマそのものや業界構造の知識がボトルネックになる。会議では CASE・SDV・Tier1・UN-R155 のような単語が前提として飛び交うが、IT 出身だとどこから手をつけていいか分かりにくい。

そんな時に「何から、どの順番で学べばいいか」を整理する。

## 結論：まず1〜2冊だけ選ぶなら

### 最初の1冊：モーターファン・イラストレーテッド

- 月刊雑誌で、1号につき1テーマ完結（EV 号／エンジン号／SDV 号など）
- 図解中心で「クルマがどう動いているか」が視覚的に頭に入る
- 1冊2,000円前後、週末で読み切れる分量
- 自分の業務に近い号（EV、電動化、SDV、コネクテッド系）を選ぶのがコスパ良し

### 2冊目：日経Automotive を1冊試す

- 業界誌の定番。**用語に一気に慣れるための投資として効く**
- 合えば3ヶ月購読することで、業界の「今」がほぼ追える
- 電子版あり

### 電子書籍で済ませたいなら

- Kindle で「自動車工学 入門」「クルマ しくみ」を検索し、評価上位の1冊
- モーターファン・イラストレーテッドも電子版あり

「クルマそのもの（メカ）」と「業界の構造」の2方向から最初に1冊ずつ入れると、効率よく土台ができる。

## 学習レイヤー全体像

### レイヤー1：クルマそのもの（メカ・しくみ）

> 物理的にクルマを理解する層

- モーターファン・イラストレーテッド（雑誌）
- 自動車工学の入門書（書店の自動車コーナーで現物を見て選ぶ）
- YouTube：自動車整備士系・自動車工学系のチャンネル

### レイヤー2：業界の構造（OEM・Tier1、CASE、ビジネス）

> 業界用語を「会話で困らない」レベルに引き上げる層

- **日経Automotive**（月刊・有料）：定番業界誌
- **日経クロステック モビリティ**（Web）：日経Automotiveのオンライン版的存在
- **藤本隆宏「能力構築競争」**：少し古いが、日本の自動車業界の構造を語る古典

### レイヤー3：SDV・コネクテッドカーのトレンド

> 仕事に直結する技術トレンド層

- 自動車技術会（JSAE）の論文・解説（無料公開分あり）
- ボッシュ・デンソー・ETAS の技術ブログ／ホワイトペーパー（Tier1 視点で SDV の実装が見える）
- TIER IV のブログ（自動運転寄り）

### レイヤー4：クラウド × 自動車（AWS視点）

> クラウドエンジニアとしての本業との接続点

- **AWS for Automotive**（公式）：リファレンスアーキテクチャ、ホワイトペーパー
- **AWS re:Invent の Automotive トラック**：YouTube 無料公開、毎年新セッションが出る
- **AWS Black Belt の Connected Vehicle 系**：日本語で短時間でインプット可

## おすすめの進め方（3〜6ヶ月）

1. **モーターファン・イラストレーテッドを書店で1〜2冊**（クルマ用語に慣れる）
2. **日経Automotive を3ヶ月購読**（業界の今と用語が一気に入る）
3. **AWS for Automotive を上から下まで読む**（本業との接続点が見える）
4. ここまで来たら CASE / SDV / UN-R155 / ISO 21434 などで検索しても抵抗なく読めるようになる

レイヤー4から入ると単語の意味が分からず手が止まる。レイヤー1から順に積むのが結局早い。
