# Agentic Loop の仕組み

## なぜこれを知っておくといいか

Claude Code に「このファイルを直して」と頼むと、Claude は勝手にファイルを読んで・考えて・書いて・確認して…と複数の操作を自律的にこなす。

「なぜ Claude がこういう動きをするのか」を理解していると、うまく頼めるようになるし、「なぜここで止まるのか」も見当がつくようになる。

---

## Agentic Loop とは

Claude Code の中核にある動作パターン。**「観察 → 思考 → 実行」を繰り返すサイクル**のこと。

```
┌──────────────────────────────┐
│                              │
│  観察（Observe）              │
│  コードを読む・状況を把握する  │
│                              │
│           ↓                  │
│                              │
│  思考（Plan）                 │
│  次に何をすべきか決める        │
│                              │
│           ↓                  │
│                              │
│  実行（Act）                  │
│  ファイル編集・コマンド実行    │
│                              │
│           ↓                  │
│                              │
│  結果を観察して次のループへ    │
│                              │
└──────────────────────────────┘
```

人間が「確認 → 判断 → 操作」を繰り返すのと同じ構造。ただし Claude は 1 リクエストの中で何十回もこのサイクルを回せる。

---

## 具体的な流れ

「`auth.py` の認証ロジックにバグがある」と伝えたとき、Claude の内部ではこんなことが起きている：

1. **観察**：`auth.py` を読む
2. **思考**：関連する `user.py` も見る必要がある
3. **実行**：`user.py` を読む
4. **観察**：`verify_token` 関数に問題を発見
5. **思考**：修正方法を決める
6. **実行**：`auth.py` を書き換える
7. **観察**：変更内容を確認
8. **思考**：テストも確認すべき
9. **実行**：テストを実行
10. **観察**：テストが通ることを確認 → 完了

これが「1 回の指示でいろいろやってくれる」理由。

---

## ループが止まるとき

Agentic Loop は以下の状況で止まる：

- **タスクが完了した**と Claude が判断したとき
- **権限のない操作**（許可していないファイル削除など）が必要になったとき → 確認を求めてくる
- **最大ターン数**に達したとき（無限ループ防止）
- ユーザーが **Esc** でキャンセルしたとき

---

## これを知って変わること

**大きな指示を 1 回で出していい**
Claude は自分でサブタスクに分解して進めてくれる。「ステップ 1 を実行して、次に…」と分割して指示しなくていい。

**「どのファイルを読むか」は Claude に任せていい**
関連ファイルは自分で探しに行く。全部手動で `@` 指定しなくてもいい。

**途中で止まるのは正常**
権限の必要な操作や不明点があると Claude はループを止めて確認してくる。これは安全装置として機能している。
