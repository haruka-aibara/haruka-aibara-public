# 出力スタイルの設定

## こういうときに使いたい

「コードを書いてもらうときは説明不要で答えだけほしいが、学習中は実装の意図も説明してほしい」「コードレビューをさせるとき、Claude の回答スタイルを統一したい」。

Claude Code はデフォルトのコーディングモード以外に、場面に合わせた**出力スタイル**に切り替えられる。

---

## 利用可能なスタイル

### Standard（デフォルト）

通常のコーディングモード。コードを書き・修正し・説明する標準的なスタイル。

### Explanatory（解説スタイル）

コードを書くだけでなく、**実装の意図・設計の判断理由**も一緒に説明してくれる教育的なスタイル。

「なぜこのアーキテクチャにしたか」「なぜこのライブラリを選んだか」をコードと合わせて解説する。

**向いている場面：**
- 知らないコードベースのキャッチアップ
- 後輩への説明資料として使いたい
- 設計の意図を残したドキュメントを作りたい

### Learning（学習スタイル）

コードを完全に書いてしまうのではなく、**`TODO(human)` マーカーを置いて手を動かすことを促す**協働スタイル。

実装の骨格は作るが、重要な部分はあえて空けて考えさせる。

**向いている場面：**
- コードを見ながら自分で実装を試したい
- チュートリアルや学習教材として使いたい
- 手を動かしながら理解を深めたい

---

## スタイルの切り替え方

### プラグインでインストールする

```bash
# Explanatory スタイル
/plugin install explanatory-output-style@claude-plugins-official

# Learning スタイル
/plugin install learning-output-style@claude-plugins-official
```

### セッション内で切り替える

インストール後、セッション内でスタイルを指定する：

```
/output-style explanatory
```

---

## CLAUDE.md でデフォルトを設定する

特定のプロジェクトで常に同じスタイルを使いたい場合は CLAUDE.md に書いておく：

```markdown
# 出力スタイル
このプロジェクトでは explanatory スタイルで回答すること。
実装の選択理由を必ずコードと一緒に説明する。
```
