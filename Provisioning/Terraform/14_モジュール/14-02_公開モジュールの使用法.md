# 公開モジュールの使用法

VPC を一から書くとサブネット・ルートテーブル・NAT Gateway・Internet Gateway だけで何十リソースにもなる。公開モジュールを使えば、実績のあるベストプラクティス構成を数行で導入できる。

---

## 基本的な書き方

```hcl
module "vpc" {
  source  = "terraform-aws-modules/vpc/aws"
  version = "~> 5.0"

  name = "my-vpc"
  cidr = "10.0.0.0/16"

  azs             = ["ap-northeast-1a", "ap-northeast-1c"]
  private_subnets = ["10.0.1.0/24", "10.0.2.0/24"]
  public_subnets  = ["10.0.101.0/24", "10.0.102.0/24"]

  enable_nat_gateway = true
}
```

---

## よく使われる公開モジュール

| モジュール | 用途 |
|---|---|
| `terraform-aws-modules/vpc/aws` | VPC・サブネット・ルートテーブル |
| `terraform-aws-modules/eks/aws` | Amazon EKS クラスター |
| `terraform-aws-modules/rds/aws` | RDS インスタンス |
| `terraform-aws-modules/s3-bucket/aws` | S3 バケット（セキュリティ設定済み） |
| `terraform-aws-modules/iam/aws` | IAM ロール・ポリシー |

---

## version の固定

```hcl
# ❌ バージョンを指定しない（危険）
module "vpc" {
  source = "terraform-aws-modules/vpc/aws"
}

# ✅ メジャーバージョンを固定
module "vpc" {
  source  = "terraform-aws-modules/vpc/aws"
  version = "~> 5.0"
}
```

バージョンを固定しないと、モジュールの更新で意図しない変更が入る可能性がある。

---

## 使用前に確認すること

1. **ソースコードを読む**: レジストリのページから GitHub に飛んで内容を確認する
2. **ライセンスを確認**: Apache 2.0 や MIT なら商用利用可
3. **メンテナンス状況**: 最終更新日・Issue の反応速度
4. **バージョン**: 安定版（v1.0+）かどうか

`terraform-aws-modules` は HashiCorp や AWS の関係者も関与する信頼性の高いモジュール群。それ以外のモジュールは内容を十分確認してから使う。
