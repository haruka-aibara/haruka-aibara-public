# ワークスペース（Workspace）

dev と prod で同じコードを使いたいが、State は分けたい——ディレクトリを複製せずにこれを実現するのがワークスペース。同一の `.tf` ファイルからワークスペース単位で独立した State を持てる。

---

## 基本的なコマンド

```bash
# ワークスペース一覧
terraform workspace list

# 新規作成
terraform workspace new staging

# 切り替え
terraform workspace select production

# 現在のワークスペース確認
terraform workspace show
```

---

## ワークスペース名の参照

`.tf` 内で現在のワークスペース名を参照できる。

```hcl
resource "aws_instance" "web" {
  instance_type = terraform.workspace == "production" ? "t3.large" : "t3.micro"

  tags = {
    Env = terraform.workspace
  }
}
```

---

## State ファイルの分離

ワークスペースごとに State が分離される。S3 バックエンドの場合：

```
s3://my-tfstate/
├── env:/
│   ├── staging/terraform.tfstate
│   └── production/terraform.tfstate
└── terraform.tfstate  # default ワークスペース
```

---

## ワークスペース vs ディレクトリ分離

| | ワークスペース | ディレクトリ分離 |
|---|---|---|
| コードの共有 | 同一の .tf を使う | コピーまたはモジュール参照 |
| 環境差異の表現 | `terraform.workspace` で分岐 | 変数ファイルで制御 |
| 独立性 | State のみ分離 | 完全に独立 |

環境差異が大きい場合はディレクトリ分離の方が管理しやすいことが多い。ワークスペースはシンプルな環境差異（インスタンスサイズ等）に向いている。
