# file プロビジョナー

EC2 を作った直後に設定ファイルやセットアップスクリプトを転送したいとき、`scp` を手動で実行する代わりに `file` プロビジョナーで Terraform から直接転送できる。`remote-exec` と組み合わせて「転送して実行」という使い方が多い。

---

## 基本的な使い方

```hcl
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"

  connection {
    type        = "ssh"
    user        = "ec2-user"
    private_key = file("~/.ssh/id_rsa")
    host        = self.public_ip
  }

  # ファイルを転送
  provisioner "file" {
    source      = "scripts/setup.sh"
    destination = "/tmp/setup.sh"
  }

  # 転送後に実行
  provisioner "remote-exec" {
    inline = [
      "chmod +x /tmp/setup.sh",
      "/tmp/setup.sh",
    ]
  }
}
```

---

## ディレクトリの転送

```hcl
provisioner "file" {
  source      = "configs/"       # ローカルのディレクトリ
  destination = "/etc/myapp"     # リモートの宛先
}
```

`source` がディレクトリの場合、中身ごと転送される。

---

## content を使ったインライン書き込み

ローカルファイルを用意せずに文字列を直接書き込める。

```hcl
provisioner "file" {
  content     = "Hello, ${self.public_ip}!"
  destination = "/tmp/hello.txt"
}
```

---

## 注意点

- SSH または WinRM の接続設定（`connection` ブロック）が必須
- Terraform 実行環境からリモートインスタンスへの疎通が必要
- CI/CD 環境では SSH ポートの開放やキー管理が課題になる
- 可能であれば S3 + cloud-init や Systems Manager でファイル配置する方がシンプル
