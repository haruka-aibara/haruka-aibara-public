# エンタープライズ機能

HCP Terraform Plus / Enterprise プランで利用できる機能。大規模・コンプライアンス要件が厳しい組織向けの機能群。

---

## SSO（シングルサインオン）

Okta・Azure AD・Google Workspace などの IdP との SAML/OIDC 連携。

- 組織の ID 管理と一元化できる
- ユーザーの Terraform アクセスを HR システムと連動させられる
- MFA をカバーできる

---

## 監査ログ

誰がいつ何をしたかを記録する。

```json
{
  "timestamp": "2024-01-15T10:30:00Z",
  "actor": "user@example.com",
  "action": "workspace.apply",
  "workspace": "production",
  "resource_count": 42,
  "status": "applied"
}
```

金融・医療など規制業界でコンプライアンス要件として必要になることが多い。

---

## Sentinel ポリシーエンジン

Plan → Sentinel チェック → Apply の強制フロー。

```python
# 本番環境への t2.micro の deploy を禁止
main = rule {
  all tfplan.resource_changes as _, rc {
    rc.type != "aws_instance" or
    rc.change.after.instance_type not in ["t2.micro", "t2.nano"]
  }
}
```

`hard-mandatory` レベルのポリシーはどのユーザーもオーバーライドできない。

---

## チームと権限管理

| ロール | できること |
|---|---|
| Owner | 組織の全設定 |
| Admin | ワークスペースの作成・設定 |
| Plan | plan のみ実行可 |
| Write | plan + apply の実行可 |
| Read | 結果の閲覧のみ |

ワークスペースごとにチームのアクセス権限を細かく設定できる。

---

## Private Registry

組織内のカスタムモジュールを Terraform Registry と同じインターフェースで提供できる。

```hcl
module "vpc" {
  source  = "app.terraform.io/my-org/vpc/aws"
  version = "~> 2.0"
}
```

社内モジュールのバージョン管理・ドキュメント公開・利用状況の把握ができる。

---

## Terraform Enterprise（セルフホスト版）

HCP Terraform を自社インフラ上にホストするバージョン。

- インターネットへの接続が許可されない環境
- データをクラウドに出せないセキュリティ要件
- 既存のプライベートネットワーク内に統合したい

Docker または Kubernetes 上で動作する。
