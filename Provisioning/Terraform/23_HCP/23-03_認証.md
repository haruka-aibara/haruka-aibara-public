# HCP Terraform の認証

HCP Terraform との認証方法と、Workspace に設定するクラウドプロバイダーの認証情報管理。

---

## HCP Terraform へのログイン

```bash
# ブラウザが開いてトークンを取得
terraform login

# トークンは ~/.terraform.d/credentials.tfrc.json に保存される
```

CI 環境では環境変数で渡す。

```bash
export TF_TOKEN_app_terraform_io="<your-token>"
```

---

## API トークンの種類

| トークン種別 | 用途 |
|---|---|
| User トークン | 個人の開発環境 |
| Team トークン | CI/CD で使う。チームの権限を持つ |
| Organization トークン | 組織全体の管理操作 |

CI では Team トークンを使うのが推奨。User トークンは個人に紐づくため、その人が退職するとトークンが無効になる。

---

## Workspace Variables への認証情報設定

HCP Terraform の Workspace に AWS 認証情報を設定する。

```
Workspace → Variables → Add variable

Key:   AWS_ACCESS_KEY_ID
Value: AKIAXXXXXXXXXXXXXXXX
☑ Sensitive  ← チェックすると値が隠れる

Key:   AWS_SECRET_ACCESS_KEY
Value: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
☑ Sensitive
```

Sensitive にした変数は UI から読めなくなり、ログにも表示されない。

---

## Dynamic Credentials（推奨）

静的な AWS キーの代わりに、OIDC を使って一時的な IAM ロールを取得する。

```hcl
# Workspace Variables に設定
TFC_AWS_PROVIDER_AUTH = true
TFC_AWS_RUN_ROLE_ARN  = "arn:aws:iam::123456789012:role/terraform-cloud-role"
```

AWS IAM ロールの Trust Policy：

```json
{
  "Effect": "Allow",
  "Principal": {
    "Federated": "arn:aws:iam::123456789012:oidc-provider/app.terraform.io"
  },
  "Action": "sts:AssumeRoleWithWebIdentity",
  "Condition": {
    "StringEquals": {
      "app.terraform.io:aud": "aws.workload.identity",
      "app.terraform.io:sub": "organization:my-org:workspace:production:run_phase:*"
    }
  }
}
```

キーのローテーションが不要になり、最小権限を Workspace 単位で付与できる。

---

## Variable Sets

複数の Workspace に同じ変数を適用したい場合は Variable Sets を使う。

```
Organization → Variable Sets → Create

Name: AWS Production Credentials
Scope: 特定の Workspace または全 Workspace

Variables:
  AWS_ACCESS_KEY_ID     = ...
  AWS_SECRET_ACCESS_KEY = ...
  AWS_DEFAULT_REGION    = ap-northeast-1
```

環境ごとの認証情報を Workspace Sets で管理すると、個別 Workspace に毎回設定しなくて済む。
