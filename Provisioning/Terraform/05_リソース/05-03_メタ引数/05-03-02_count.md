# count

「EC2 を 3 台作りたい」「環境変数で台数を変えたい」——同じリソースを複数まとめて作りたいときに使うのが `count`。ただし途中の要素を削除するとインデックスがずれて意図しない replace が起きるため、名前で識別したい場合は `for_each` の方が安全。

---

## 基本的な使い方

```hcl
resource "aws_instance" "web" {
  count         = 3
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"

  tags = {
    Name = "web-${count.index}"  # 0, 1, 2
  }
}
```

`count.index` で現在のインデックス（0始まり）を参照できる。

---

## 変数と組み合わせる

```hcl
variable "instance_count" {
  type    = number
  default = 2
}

resource "aws_instance" "web" {
  count = var.instance_count
  ...
}
```

---

## 生成されたリソースの参照

```hcl
# 全インスタンスのIDをリストで取得
output "instance_ids" {
  value = aws_instance.web[*].id
}

# 特定のインスタンスを参照
resource "aws_eip" "first" {
  instance = aws_instance.web[0].id
}
```

---

## count vs for_each

| | count | for_each |
|---|---|---|
| インデックス | 数値（0, 1, 2...） | map/set のキー |
| リソースID | `aws_instance.web[0]` | `aws_instance.web["名前"]` |
| 途中削除 | インデックスがずれる（要注意） | キーで管理するためずれない |

途中のリソースを削除する可能性がある場合は `for_each` を使う方が安全。
