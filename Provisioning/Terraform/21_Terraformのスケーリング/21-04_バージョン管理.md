# バージョン管理

Terraform 本体とプロバイダーのバージョンを固定することは、チーム全員が同じ動作をする環境を保つために必須。

---

## Terraform 本体のバージョン固定

`required_version` で許可するバージョンを制約する。

```hcl
terraform {
  required_version = "~> 1.7.0"   # 1.7.x のみ許可

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}
```

`~> 1.7.0` の意味：1.7.0 以上 1.8.0 未満（パッチバージョンの更新のみ許可）。

---

## tfenv でバージョンを切り替える

プロジェクトによって Terraform のバージョンが異なる場合に使う。

```bash
# インストール
brew install tfenv

# 利用可能なバージョン確認
tfenv list-remote

# バージョンを指定してインストール
tfenv install 1.7.5

# プロジェクトで使うバージョンを指定
echo "1.7.5" > .terraform-version
tfenv use 1.7.5
```

`.terraform-version` ファイルをリポジトリにコミットしておくと、チーム全員が同じバージョンを使える。

---

## プロバイダーのバージョンロック

`terraform init` を実行すると `.terraform.lock.hcl` が生成される。これをコミットする。

```
# .terraform.lock.hcl（自動生成）
provider "registry.terraform.io/hashicorp/aws" {
  version     = "5.31.0"
  constraints = "~> 5.0"
  hashes = [
    "h1:abc123...",
    # ...
  ]
}
```

```bash
# ロックファイルをコミット対象にする
git add .terraform.lock.hcl
```

ロックファイルがあることで、チーム全員・CI が同じプロバイダーバージョンを使う。

---

## プロバイダーのアップデート手順

```bash
# バージョン制約を満たす最新バージョンに更新
terraform init -upgrade

# ロックファイルの差分を確認してコミット
git diff .terraform.lock.hcl
git add .terraform.lock.hcl
git commit -m "chore: upgrade aws provider to 5.x"
```

アップデート後は `terraform plan` で予期しない差分がないか確認する。

---

## CI でのキャッシュ

毎回 `terraform init` でプロバイダーをダウンロードすると遅い。CI でキャッシュを使う。

```yaml
# GitHub Actions の例
- uses: actions/cache@v3
  with:
    path: ~/.terraform.d/plugin-cache
    key: terraform-${{ hashFiles('**/.terraform.lock.hcl') }}
```
