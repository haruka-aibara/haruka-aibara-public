# terraform init

`.tf` ファイルを書いたら必ず最初に実行するコマンド。これを実行しないと `plan` も `apply` も動かない。プロバイダープラグインのダウンロードとバックエンドの初期化を行う。

---

## 実行内容

```bash
terraform init
```

1. `required_providers` に書かれたプロバイダープラグインを `.terraform/` にダウンロード
2. バックエンド（S3 等）の設定を初期化
3. `terraform.lock.hcl` を生成・更新

---

## `.terraform/` と `.gitignore`

`init` で作られるプロバイダーバイナリの保存場所。サイズが大きいので Git には入れない。

```gitignore
.terraform/
*.tfstate
*.tfstate.backup
```

---

## `terraform.lock.hcl` はコミットする

プロバイダーの実際のバージョンを固定するロックファイル。**これは Git にコミットする**。

```hcl
provider "registry.terraform.io/hashicorp/aws" {
  version     = "5.31.0"
  constraints = "~> 5.0"
  hashes = ["h1:..."]
}
```

コミットしておくと、チームメンバーや CI が `terraform init` したとき全員同じバージョンのプロバイダーを使うことが保証される。

---

## よく使うオプション

```bash
# プロバイダーを最新にアップグレード（lock.hcl も更新される）
terraform init -upgrade

# バックエンドを変更して再初期化
terraform init -reconfigure

# バックエンド設定を CI/CD から動的に渡す
terraform init -backend-config="bucket=my-tfstate-bucket"
```
