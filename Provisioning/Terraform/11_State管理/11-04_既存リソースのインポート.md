# 既存リソースのインポート

「今まで手動でコンソールから作ってた EC2 を Terraform 管理に移したい」「前任者が CloudFormation で作ったリソースを Terraform に移行したい」——こういう場面がある。`terraform import` を使えば、既存のリソースを Terraform の State に取り込んで管理下に置ける。

---

## import ブロック（v1.5+ 推奨）

`.tf` に宣言的に書いて `apply` でインポートできる。

```hcl
# インポートする対象を宣言
import {
  to = aws_instance.web
  id = "i-0abc123def456789"   # AWS のリソース ID
}

# 対応するリソース定義も必要
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"
}
```

`terraform plan` で差分を確認してから `terraform apply`。

---

## `-generate-config-out` で定義を自動生成（v1.5+）

既存リソースの `.tf` を手で書くのが大変なときは、自動生成を活用する。

```bash
terraform plan -generate-config-out=generated.tf
```

生成されたファイルをベースに修正すると楽。すべての属性が正確に生成されるとは限らないので、プロバイダーのドキュメントと照合して確認する。

---

## terraform import コマンド（旧来の方法）

```bash
terraform import aws_instance.web i-0abc123def456789
```

State にリソースを追加するだけ。対応する `.tf` ファイルの定義は別途手で書く必要がある。

---

## インポート後は必ず plan で確認する

```bash
terraform plan
```

差分が出た場合は `.tf` の定義と実際のリソースの設定がずれている。定義を直すか、変動する属性は `ignore_changes` で無視する。差分がゼロになって初めてインポート完了。
