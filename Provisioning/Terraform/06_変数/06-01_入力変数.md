# 入力変数

同じ Terraform コードを dev・staging・prod で使い回したい。でも環境によってインスタンスサイズや設定値が違う。コードに直接 `"t3.micro"` と書いてしまうと、環境ごとにコードをコピーするか毎回書き換えるしかなくなる。

それを解決するのが変数。環境差分を変数として切り出してコードを共通化する。

---

## variable ブロック

```hcl
variable "instance_type" {
  type        = string
  default     = "t3.micro"
  description = "EC2 インスタンスタイプ"
}

variable "env" {
  type = string
  # default なし → apply 時に必ず指定が必要な必須変数
}
```

---

## 使い方

```hcl
resource "aws_instance" "web" {
  instance_type = var.instance_type
  tags = {
    Environment = var.env
  }
}
```

---

## 値の渡し方

**コマンドライン**（一時的な上書きに使う）
```bash
terraform apply -var="env=production" -var="instance_type=t3.large"
```

**変数定義ファイル**（環境ごとに用意するのが一般的）
```bash
terraform apply -var-file="production.tfvars"
```

`production.tfvars`:
```hcl
env           = "production"
instance_type = "t3.large"
```

**環境変数**（CI/CD で使いやすい）
```bash
export TF_VAR_env="production"
terraform apply
```

`TF_VAR_` プレフィックスをつけると自動で変数として読み込まれる。

---

## 優先順位（高い順）

複数の方法で同じ変数を指定したとき、どれが勝つか：

1. `-var` コマンドライン引数
2. `-var-file` で指定したファイル
3. `*.auto.tfvars`（自動読み込み）
4. `terraform.tfvars`（自動読み込み）
5. 環境変数 `TF_VAR_*`
6. `default` 値

CI でデフォルト値を上書きしたいときは `-var` か環境変数が手軽。
