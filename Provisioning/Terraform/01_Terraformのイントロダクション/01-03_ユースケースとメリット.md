# Terraform のユースケースとメリット

---

## 主なユースケース

### マルチクラウド管理

AWS と GCP を同じ HCL で管理できる。クラウド固有のツール（CloudFormation 等）と違い、記法・ワークフローが統一される。

```hcl
resource "aws_instance" "web" { ... }
resource "google_compute_instance" "app" { ... }
```

### 環境の再現

`terraform apply` で dev / staging / prod を同一コードから作成できる。`terraform.tfvars` で環境ごとに変数を切り替えるだけ。

### インフラの Gitflow

インフラ変更を PR → レビュー → merge → apply の流れで管理できる。意図しない変更を防ぎ、変更履歴が残る。

### 既存リソースのコード化

`terraform import` で手動作成済みのリソースを Terraform 管理下に取り込める。v1.5 以降では `import` ブロックで宣言的に書ける。

---

## メリット

- **冪等性**: 何度 apply しても同じ結果になる
- **差分管理**: `terraform plan` で変更内容を事前確認できる
- **モジュール化**: 共通パターンをモジュールとして再利用できる
- **エコシステムの豊富さ**: レジストリに数千のプロバイダー・モジュール
