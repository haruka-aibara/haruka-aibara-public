# 1か月で受かる勉強ルートと、その先の活用に繋げ方

## こういう場面のための記事

「生成AIを業務で使えるようになりたい。でも、何となく ChatGPT に質問してるだけで、体系的にわかってる気がしない」「資格があれば社内提案や転職時に話のとっかかりになる」「ただし、半年も準備に使うのは現実的じゃない」

そんな状況で「1か月くらいで受かるなら受けてみよう」と動き始める人向け。生成AIパスポートは合格率 75〜80% 前後で、難易度は「真面目にやれば落ちない」レベル。だからこそ、最短ルートと「合格しても使えない知識止まり」にしないやり方の両方を押さえておきたい。

## まず試験のスペックを把握する

ここを誤解していると勉強配分を間違える。

| 項目 | 内容 |
|---|---|
| 主催 | 一般社団法人 生成AI活用普及協会（GUGA） |
| 形式 | IBT（自宅PC・スマホ・タブレットで受験） |
| 試験時間 | 60分 |
| 問題数 | 60問（四肢択一、一部複数選択） |
| 受験料 | 一般 11,000円 / 学生 5,500円（税込） |
| 開催頻度 | 年5回（2月・4月・6月・8月・10月） |
| 受験期間 | 各回1か月（その月の1日〜末日の任意の時間） |
| 合格率 | 78.84%（2026年2月回） |

特徴的なのは「**受験期間が1か月**」あること。申込みすれば1か月のなかで好きな時間に受験できる。だから「受験日が動かせない」というプレッシャーがない反面、「いつでもやれる」と思って先延ばしになりがち。先に申込み日を決めて、試験予定日を逆算するほうがいい。

## シラバスは2026年2月に大改訂された

ここが今このタイミングで受ける人にとって最重要ポイント。古い対策本・古いブログ記事をそのまま信じると痛い目に遭う。

2026年2月試験から、以下が**追加された**：

- **第2章**：GPT-o1 / o3 / o4 / GPT-4.1 / GPT-5 / Sora / Operator / Codex / Image Generation / Gemini / Claude / Copilot
- **第3章**：RAG（チャンク・ベクトルデータベース含む）、AIエージェント（GenSpark / Manus / Skywork AI / MCP）、Veo3
- **第4章**：AI事業者ガイドライン第1.1版への更新、AI新法（人工知能関連技術の研究開発及び活用の推進に関する法律、2025年6月公布）

特に **RAG・AIエージェント・MCP・AI新法** は古い教材だとまったく載っていない。ここを取りこぼすと60問のうち5〜10問落とす可能性がある。

## 出題範囲の全体像（5章構成）

| 章 | テーマ | ボリューム感 | 性格 |
|---|---|---|---|
| 第1章 | AI（人工知能）の定義・歴史・仕組み | 中 | 暗記中心。ダートマス会議〜ディープラーニング、過学習、転移学習など |
| 第2章 | 生成AIモデル（GPT系の系譜） | 大 | キーワードが多い。Transformer・GPT各バージョン・他社モデル |
| 第3章 | 現在の動向（画像/動画/音楽生成、RAG、AIエージェント） | 大 | 改訂で重くなった章。最新トピック中心 |
| 第4章 | 法律・倫理・ガイドライン | 大 | 著作権・個人情報・AI社会原則・AI新法。配点高い印象 |
| 第5章 | プロンプト制作と実例 | 中 | 実務寄り。Zero-Shot / Few-Shot、ユースケース、不得意なこと |

肌感として **第2章・第3章・第4章で6〜7割を占める**。第1章は基礎知識、第5章は常識的な内容で済むので、対策の重心は中盤3章に置く。

## 1か月の勉強ルート

合計学習時間の目安は **15〜25時間**。1日30〜60分でも届く。週ごとの組み立てはこう。

### 第1週：全体像とAIの基礎（第1〜2章）

- 公式テキストを通読する（読み流しでOK、ここで覚えようとしない）
- 第1章の用語（教師あり/なし学習、強化学習、過学習、転移学習）を押さえる
- 第2章の **GPTの系譜を年表で整理する**：GPT-1 → GPT-2 → GPT-3 → InstructGPT → GPT-3.5 → GPT-4 → GPT-4o → o1 → o3 → o4 → GPT-4.1 → GPT-5

GPTのバージョン違いは丸暗記より「何が新しくなったか」をメモしながら読むと残る。たとえば InstructGPT は RLHF（人間のフィードバックによる強化学習）を入れたことで対話品質が上がった、など。

### 第2週：最新トピックを潰す（第3章）

ここが**最大の山場**。改訂の追加分が集中している。

- **RAG**：なぜ必要か（LLM単体だと社内文書や最新情報を答えられない）→ どう動くか（チャンク化 → ベクトルDB に格納 → 検索 → LLMに渡して回答生成）→ ユースケース（社内QA、カスタマーサポート、ドキュメント検索）
- **AIエージェント**：従来のチャットボットとの違いは「自分でタスクを分解して、ツールを使って完了まで動く」こと。GenSpark / Manus / Skywork AI が代表例
- **MCP（Model Context Protocol）**：AIエージェントが外部ツール・データソースに繋がるための共通プロトコル。Claude を提供する Anthropic が提唱
- **画像/動画生成**：Sora、Veo3、Image Generation の位置づけを押さえる

ここは古い教材に載っていないので、**公式シラバスの詳細キーワードをそのまま検索して、各社の公式ページで一次情報を読む**のが速い。

### 第3週：法律・倫理（第4章）と問題演習開始

第4章は配点が大きい。法律系は混同しやすいので整理表を作る。

| 守る対象 | 関連する権利・法律 |
|---|---|
| 創作物 | 著作権 |
| 発明 | 特許権 |
| ブランド名・ロゴ | 商標権 |
| 製品デザイン | 意匠権 |
| 顔・容姿 | 肖像権、パブリシティ権（著名人の経済的価値） |
| 営業秘密・限定提供データ | 不正競争防止法 |
| 個人情報 | 個人情報保護法（要配慮個人情報・匿名加工情報） |

**AI新法（2025年6月4日公布）** は新規追加の超重要論点。「規制法ではなく、研究開発と活用の推進が目的」という性格を押さえる。AI事業者ガイドラインとセットで出る可能性が高い。

AI事業者は3つの主体（**AI開発者 / AI提供者 / AI利用者**）に分類されることも頻出。

このタイミングで **無料の問題集や模擬問題で1周目を回す**。間違えたところを集中的にテキストで補強する。

### 第4週：問題演習を繰り返して仕上げ（第5章含む）

- 第5章は通読すれば足りる。Zero-Shot / Few-Shot プロンプティングの違い、プロンプトの構成要素（Instruction / Context / Input Data / Output Indicator）、テキスト生成AIの**不得意なこと**（正確な文字数指定、計算、最新情報、芸術の批評）は出題されやすい
- 問題集を2〜3周。**間違えた問題だけを集めたノート**を作って、試験直前に見返せる状態にする
- 模擬試験で60分・60問のペース感を体に入れる

## 教材の選び方

| 教材 | 性格 | 必要性 |
|---|---|---|
| 公式テキスト（GUGA公認）「生成AIパスポート テキスト&問題集」最新版 | 試験範囲を体系的にカバー。改訂対応版を選ぶ | 必須に近い |
| 無料の過去問サイト（「過去問ふぅ問題集」など） | 数百問規模、解説付き | 強くおすすめ |
| 試験対策アプリ（トレ速、KANTAN系） | スキマ時間用 | 通勤時間がある人は◎ |

ポイントは **「2026年2月以降の改訂シラバス対応」と明示されているもの**を選ぶこと。Amazon で買うときも発行年と版数を必ず確認する。

## 試験当日のクセ

- IBTなので**自宅環境**で受ける。安定したネット回線・静かな部屋・本人確認書類を準備
- **60分で60問 = 1問1分**。長文の選択肢を読み込みすぎない
- **複数選択問題が混ざる**ので、「これは1つだけ選ぶのか複数か」を必ず確認
- 受験期間中ならいつでも受けられるが、期限ギリギリにしない（システムトラブル時の予備日を残す）

## ここからが本題：合格しても「使える」とは限らない

生成AIパスポートはあくまで**用語と概念のリテラシー試験**。プロンプトを書ける、業務に組み込める、トラブルシューティングできる、ことは保証しない。

合格を**スタートライン**にして、ここから3つの方向に手を伸ばすと「使える側」になれる。

### 方向1：自分の業務で1つ「定着させる」

「ChatGPTで何ができるか」ではなく、「**自分の毎週やってる定型業務のうち、どれを置き換えるか**」を決める。例：

- 会議議事録の要約 → 録音 → 文字起こし → ChatGPT/Claude で要約・タスク抽出のパイプラインを作る
- メール下書き → 自分のトーンに合わせた指示プロンプトをテンプレ化
- リサーチ → Perplexity / GenSpark で一次調査 → ChatGPT で構造化

**1個でいいから定着させる**。「使えそう」と「使ってる」の差が、資格保持者と実務者の差。

### 方向2：プロンプトエンジニアリングを一段深く

試験で出る Zero-Shot / Few-Shot は入口に過ぎない。実務では：

- **Chain-of-Thought**（思考過程を明示させる）
- **ロール設定**（「あなたは経験10年のレビュアーです」）
- **構造化出力**（JSON / Markdown / 表で返させる）
- **ステップバイステップで指示を分解**

このあたりを試行錯誤すると、出力品質が体感で1段階上がる。

### 方向3：RAG・AIエージェントを実際に触る

第3章で学んだ概念を**手を動かして確認する**。

- RAG：自分のメモやドキュメントを ChatGPT のカスタムGPTや Claude の Projects 機能に読み込ませて検索させる
- AIエージェント：GenSpark や Manus を使ってリサーチタスクを丸投げしてみる
- MCP：Claude Desktop で MCPサーバーを動かして、外部ツール連携を試す

**触ってみると「資格で覚えた言葉と、実物の挙動」が結びつく**。これが起きないと、知識が他人事のままになる。

### 方向4（おまけ）：次の資格を見据える

生成AIパスポートは「ユーザー向け」の入門資格。次に進むなら：

- **G検定（JDLA）**：もう少しディープラーニング寄りの体系的な知識
- **E資格（JDLA）**：実装まで踏み込む。エンジニア向け
- **クラウドベンダー系のAI/ML資格**（AWS Certified AI Practitioner、Azure AI Fundamentals など）：実務での選定や設計に直結する

ただ、資格を積む前に **「方向1〜3で1つ業務を変えた」実績**を作るほうが転職・社内提案では強い。資格は名刺代わり、実績が中身。

## まとめ

- **1か月・15〜25時間**で合格は十分射程内
- 改訂内容（**RAG・AIエージェント・MCP・AI新法**）が新教材で対応されているか必ず確認
- 第2〜4章に時間を厚めに、第1・5章は通読＋演習で十分
- 合格後は「**自分の業務を1つ置き換える**」「実際にツールを触る」をやらないと、合格証だけが残る
