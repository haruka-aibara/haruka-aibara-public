# templatefile 関数

`templatefile` は外部ファイルをテンプレートとして読み込み、変数を展開して文字列を生成する関数。User Data や設定ファイルの動的生成によく使う。

---

## 基本構文

```hcl
templatefile(path, vars)
```

- `path`: テンプレートファイルのパス（`.tftpl` 拡張子を推奨）
- `vars`: テンプレート内で使う変数のマップ

---

## 使用例：EC2 の User Data

`scripts/init.sh.tftpl`:

```bash
#!/bin/bash
echo "Setting up ${app_name} on ${environment}"

yum update -y
yum install -y ${package_name}

cat > /etc/myapp/config.json <<EOF
{
  "db_host": "${db_host}",
  "db_port": ${db_port}
}
EOF

systemctl start myapp
```

`main.tf`:

```hcl
resource "aws_instance" "app" {
  ami           = data.aws_ami.amazon_linux.id
  instance_type = "t3.small"

  user_data = templatefile("scripts/init.sh.tftpl", {
    app_name     = var.app_name
    environment  = var.environment
    package_name = "myapp"
    db_host      = aws_db_instance.main.address
    db_port      = aws_db_instance.main.port
  })
}
```

---

## テンプレート内で使える構文

### 条件分岐

```
%{ if environment == "production" }
  LOG_LEVEL=warn
%{ else }
  LOG_LEVEL=debug
%{ endif }
```

### ループ

```
%{ for server in servers }
  - ${server}
%{ endfor }
```

HCL の `for` 式と同じイメージで書ける。

---

## heredoc との比較

```hcl
# ❌ インラインで長い User Data を書くと管理しにくい
user_data = <<-EOF
  #!/bin/bash
  echo "Setting up ${var.app_name}..."
  # ...100行のスクリプト...
EOF

# ✅ テンプレートファイルに分離して管理
user_data = templatefile("scripts/init.sh.tftpl", {
  app_name = var.app_name
})
```

スクリプトが長くなるほど `templatefile` の方が可読性が上がる。

---

## ファイル拡張子について

`.tftpl` が公式推奨。エディタのシンタックスハイライトが対応していない場合は `.sh.tftpl` のように二重拡張子にするのが一般的。
