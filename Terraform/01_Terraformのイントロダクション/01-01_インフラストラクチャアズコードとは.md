# インフラストラクチャアズコード（IaC）とは

インフラの構成をコードで定義・管理する手法。サーバー、ネットワーク、データベースなどの設定をコードとして記述し、バージョン管理・自動化・再現を可能にする。

---

## 手動管理 vs IaC

| 手動管理 | IaC |
|---|---|
| GUI やコンソールで設定 | コードで定義 |
| 設定内容がドキュメントに残らない | Git で履歴管理 |
| 再現が難しい | `terraform apply` で同じ環境を再現 |
| 人的ミスが起きやすい | レビュー・自動テストが可能 |

---

## IaC のメリット

**再現性**: 同じコードから同じ環境を何度でも作れる。dev / staging / prod の環境差異が減る。

**バージョン管理**: インフラ変更を Pull Request でレビューできる。問題があれば git revert で戻せる。

**ドキュメントとしてのコード**: コードそのものが構成の仕様書になる。

**自動化**: CI/CD と組み合わせることで、コードの変更をトリガーにインフラを自動更新できる。

---

## 主な IaC ツール

| ツール | 特徴 |
|---|---|
| **Terraform** | クラウド非依存、HCL で記述、最も普及 |
| AWS CloudFormation | AWS 専用、YAML/JSON |
| Pulumi | TypeScript / Python 等の汎用言語で記述 |
| Ansible | 構成管理寄り、サーバー設定向け |

マルチクラウド・汎用性を重視するなら Terraform が最有力候補。
