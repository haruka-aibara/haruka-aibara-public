# VCS 統合

HCP Terraform と GitHub などの VCS（バージョン管理システム）を連携し、PR や push をトリガーに plan / apply を自動実行する。

---

## 連携できる VCS

- GitHub / GitHub Enterprise
- GitLab / GitLab EE
- Bitbucket Cloud / Bitbucket Server
- Azure DevOps

---

## セットアップ手順

1. **Organization Settings → VCS Providers** で VCS を登録
   - GitHub OAuth アプリを作成してクライアント ID・シークレットを設定
2. **Workspace の Settings → Version Control** でリポジトリを選択
3. どのブランチをトリガーにするかを設定

---

## 動作フロー

```
開発者が PR を作成
↓
HCP Terraform が自動で terraform plan を実行
↓
PR に plan 結果がコメントされる
↓
レビュー・承認
↓
main にマージ
↓
HCP Terraform が terraform apply を実行（設定次第）
```

---

## 設定例

Workspace の VCS 設定項目：

| 設定 | 内容 |
|---|---|
| Repository | `my-org/infra-repo` |
| Branch | `main`（apply のトリガー） |
| Terraform Working Directory | `terraform/production/` |
| Auto Apply | main マージ後に自動 apply するか |
| Trigger Patterns | `terraform/production/**` |

Working Directory を指定することで、モノレポの特定ディレクトリだけ対象にできる。

---

## VCS 連携 vs API-Driven Workflow

| 方式 | 向いているケース |
|---|---|
| VCS 連携 | コードを push するだけで動かしたいシンプルな運用 |
| API-Driven | 既存の GitHub Actions など CI に組み込みたい場合 |

API-Driven では CI から HCP Terraform の API を叩いて run を作成する。

```bash
# API で run を作成する例
curl -X POST \
  -H "Authorization: Bearer $TF_TOKEN" \
  -H "Content-Type: application/vnd.api+json" \
  https://app.terraform.io/api/v2/runs \
  -d '{"data": {"type": "runs", "relationships": {"workspace": {"data": {"type": "workspaces", "id": "ws-xxxxx"}}}}}'
```

## Speculative Plan

PR を出したタイミングで `speculative plan`（参考 plan）を実行する。main のコードには影響を与えず、変更の差分だけを確認できる。

PR にコメントとして表示されるため、レビュアーがインフラの変更内容を把握しやすくなる。
