# HCP Terraform のワークスペース

HCP Terraform のワークスペースは Terraform の実行環境単位。State ファイル・変数・実行履歴・権限を管理する。

---

## ワークスペースの役割

```
Organization: my-company
├── Workspace: network-dev
│   ├── State: dev 環境の VPC・サブネット
│   ├── Variables: dev 用の認証情報・変数
│   └── 実行履歴
├── Workspace: network-prod
│   ├── State: prod 環境の VPC・サブネット
│   └── ...
└── Workspace: app-prod
    └── ...
```

1 Workspace = 1 State ファイルに対応。環境・コンポーネント単位で分ける。

---

## Workspace の作成

UI または CLI から作成できる。

```bash
# CLI でワークスペース作成
terraform workspace new production
```

HCP Terraform との接続設定：

```hcl
terraform {
  cloud {
    organization = "my-company"
    workspaces {
      name = "app-prod"   # Workspace 名を直接指定
    }
  }
}
```

---

## タグによる Workspace 選択

複数の Workspace に同じコードを適用する場合。

```hcl
terraform {
  cloud {
    organization = "my-company"
    workspaces {
      tags = ["app", "production"]   # タグで複数 Workspace を選択
    }
  }
}
```

`terraform workspace select` でどの Workspace を使うか切り替える。

---

## 実行モード

| モード | 内容 |
|---|---|
| Remote | HCP Terraform のクラウド環境で実行（デフォルト） |
| Local | ローカルで実行。State だけ HCP に保存 |
| Agent | 自社ネットワーク内のエージェントで実行 |

プライベートネットワーク内のリソースを管理する場合は Agent モードを使う。

---

## 実行の承認フロー

Remote モードでは UI 上で plan 結果を確認してから apply を承認できる。

```
terraform apply
↓
HCP Terraform で plan 実行
↓
UI または CLI で確認 → "Confirm & Apply" ボタン
↓
apply 実行
```

CI から自動 apply する場合は `TF_CLI_ARGS_apply=-auto-approve` を設定するか、Workspace の "Auto apply" を有効にする。

---

## Workspace 間の State 参照

```hcl
data "terraform_remote_state" "network" {
  backend = "remote"
  config = {
    organization = "my-company"
    workspaces = {
      name = "network-prod"
    }
  }
}

# network-prod Workspace の outputs を参照
locals {
  vpc_id = data.terraform_remote_state.network.outputs.vpc_id
}
```
