# タスク実行（Run Tasks）

「Checkov で警告が出ても開発者が無視して apply してしまう」「コスト増加の PR がレビューを通り抜けてしまう」——これを防ぎたいなら、apply の手前にチェックを強制できる Run Tasks を使う。外部ツール（Infracost・Snyk・Checkov など）を HCP Terraform のパイプラインに組み込んで、通過しないと apply できない仕組みを作れる。

---

## Run Tasks の位置づけ

```
Plan → Run Tasks → Apply
            ↓
    ✓ Pass → Apply を続行
    ✗ Fail → Apply を中断
```

plan 結果を外部サービスに送信し、その結果によって apply を許可・拒否する。

---

## 利用可能な統合ツール

HCP Terraform Marketplace に公開されているもの：

| ツール | 用途 |
|---|---|
| Infracost | コスト見積もり |
| Snyk | セキュリティスキャン |
| Checkov | IaC セキュリティ |
| Bridgecrew | コンプライアンスチェック |
| HashiCorp Vault | シークレット検証 |

---

## セットアップ手順

1. **Organization Settings → Run Tasks** でタスクを登録
   - タスク名
   - エンドポイント URL（外部サービスの Webhook URL）
   - HMAC キー（署名検証用）

2. **Workspace Settings → Run Tasks** でタスクをアタッチ
   - どのタスクを実行するか
   - Enforcement Level を設定

---

## Enforcement Level

| レベル | 挙動 |
|---|---|
| Advisory | 失敗してもログに警告だけ。apply は続行 |
| Mandatory | 失敗したら apply を中断 |

開発環境では Advisory、本番環境では Mandatory にするのが一般的。

---

## カスタム Run Task の作成

独自の外部サービスを Run Task として組み込める。Webhook を受け取り結果を返す HTTP サーバーを実装する。

```
1. HCP Terraform が plan 後に Webhook を送信
   POST https://your-service.example.com/run-task
   {
     "payload_version": 1,
     "stage": "post_plan",
     "plan": { ... },
     "callback_url": "https://app.terraform.io/api/v2/task-results/..."
   }

2. 外部サービスが処理後に結果を返す
   PATCH <callback_url>
   {
     "data": {
       "attributes": {
         "status": "passed",   // または "failed"
         "message": "チェック通過"
       }
     }
   }
```

---

## 実務での使い方

本番環境への apply 前に：

1. **Infracost** で月次コストが閾値（例：$1000）を超えないか確認
2. **Checkov** でセキュリティルール違反がないか確認
3. **Sentinel** でコンプライアンスポリシーをチェック

これらを Mandatory にすることで、セキュリティ・コスト・コンプライアンスの基準を満たさない変更が本番に入らないことを保証できる。
