# 単体テスト

モジュールを書いたあと、本当に期待した設定になっているか apply してから初めて気づくのは遅い。`terraform test` を使えば、実際のインフラを作らずに plan レベルで属性やタグの検証ができる。

---

## terraform test コマンド（Terraform 1.6+）

Terraform 1.6 から `terraform test` コマンドが正式導入された。`.tftest.hcl` ファイルにテストを書く。

### ディレクトリ構成

```
modules/
└── networking/
    ├── main.tf
    ├── variables.tf
    ├── outputs.tf
    └── tests/
        └── unit.tftest.hcl
```

### テストファイルの書き方

```hcl
# tests/unit.tftest.hcl

variables {
  vpc_cidr = "10.0.0.0/16"
  env      = "test"
}

run "verify_vpc_cidr" {
  command = plan   # plan だけ実行（実インフラを作らない）

  assert {
    condition     = aws_vpc.main.cidr_block == var.vpc_cidr
    error_message = "VPC の CIDR が想定と違います"
  }
}

run "verify_tags" {
  command = plan

  assert {
    condition     = aws_vpc.main.tags["Environment"] == var.env
    error_message = "Environment タグが設定されていません"
  }
}
```

---

## 実行方法

```bash
terraform test
```

特定のファイルだけ実行：

```bash
terraform test -filter=tests/unit.tftest.hcl
```

---

## command = plan vs apply

| command | 挙動 | コスト |
|---|---|---|
| `plan` | 差分計算のみ・インフラ作成なし | 無料（API コールあり） |
| `apply` | 実際にリソースを作成・テスト後削除 | 課金対象になる場合あり |

単体テストでは `command = plan` を使ってリソースを作らずに検証するのが基本。

---

## 静的解析ツールとの組み合わせ

`terraform validate` と `tflint` は実行コストゼロで型チェック・ルール違反を検出できる。`terraform test` と組み合わせると効果的。

```bash
terraform validate
tflint
terraform test
```
