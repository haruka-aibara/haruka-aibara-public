# 変数の検証ルール（validation）

`variable` ブロックに `validation` を追加して、入力値のカスタムバリデーションを定義できる。型チェックで捕捉できない条件を強制する。

---

## 基本的な書き方

```hcl
variable "instance_type" {
  type = string

  validation {
    condition     = contains(["t3.micro", "t3.small", "t3.medium"], var.instance_type)
    error_message = "instance_type は t3.micro / t3.small / t3.medium のいずれかを指定してください。"
  }
}
```

`condition` が `false` の場合に `error_message` が表示される。

---

## 複数の検証ルール

`validation` ブロックは複数書ける。

```hcl
variable "bucket_name" {
  type = string

  validation {
    condition     = length(var.bucket_name) >= 3
    error_message = "バケット名は 3 文字以上にしてください。"
  }

  validation {
    condition     = length(var.bucket_name) <= 63
    error_message = "バケット名は 63 文字以下にしてください。"
  }

  validation {
    condition     = can(regex("^[a-z0-9][a-z0-9-]*[a-z0-9]$", var.bucket_name))
    error_message = "バケット名は小文字英数字とハイフンのみ使用できます。"
  }
}
```

---

## よく使う検証パターン

```hcl
# 正の数のみ許可
validation {
  condition     = var.instance_count > 0
  error_message = "instance_count は 1 以上を指定してください。"
}

# CIDR ブロックの形式チェック
validation {
  condition     = can(cidrhost(var.cidr_block, 0))
  error_message = "有効な CIDR ブロックを指定してください（例: 10.0.0.0/16）。"
}

# 環境名の制限
validation {
  condition     = contains(["dev", "staging", "production"], var.env)
  error_message = "env は dev / staging / production のいずれかを指定してください。"
}
```
