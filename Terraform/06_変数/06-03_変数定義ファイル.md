# 変数定義ファイル（.tfvars）

dev / staging / prod で違う値を使うとき、`-var instance_type=t3.micro` を毎回手打ちしていると漏れが出る。環境ごとの値を `.tfvars` ファイルにまとめておけば、`-var-file` の一発切り替えで済む。

---

## terraform.tfvars

`terraform apply` 実行時に自動的に読み込まれる。

```hcl
# terraform.tfvars
env           = "development"
instance_type = "t3.micro"
instance_count = 1
```

---

## *.auto.tfvars

`*.auto.tfvars` という名前のファイルも自動読み込みされる。

```hcl
# common.auto.tfvars
region = "ap-northeast-1"
tags = {
  Project = "myapp"
}
```

---

## 環境別ファイルでの管理

環境によってファイルを切り替える運用が多い。

```
envs/
├── dev.tfvars
├── staging.tfvars
└── production.tfvars
```

```bash
# 開発環境
terraform apply -var-file="envs/dev.tfvars"

# 本番環境
terraform apply -var-file="envs/production.tfvars"
```

---

## .gitignore への追加

`terraform.tfvars` に機密情報（API キー等）を書く場合は `.gitignore` に追加する。

```gitignore
# 機密情報が含まれる場合
*.tfvars
!example.tfvars  # サンプルファイルは除外しない
```

機密情報は環境変数（`TF_VAR_*`）や Secrets Manager から渡すのが安全。
