# ローカル値（locals）

複数のリソースに同じタグを付けたい。名前のプレフィックスを統一したい。でも毎回同じ値を書いていると、変更のたびに全部直す必要がある。

`locals` に共通値をまとめておけば一箇所を直すだけで済む。

---

## locals ブロック

```hcl
locals {
  app_name    = "myapp"
  env         = var.env
  name_prefix = "${local.app_name}-${local.env}"

  common_tags = {
    Project     = local.app_name
    Environment = local.env
    ManagedBy   = "Terraform"
  }
}
```

参照するには `local.名前`（`locals` ではなく `local`）。

---

## 使い方

```hcl
resource "aws_instance" "web" {
  instance_type = "t3.micro"

  tags = merge(local.common_tags, {
    Name = "${local.name_prefix}-web"
  })
}

resource "aws_s3_bucket" "logs" {
  bucket = "${local.name_prefix}-logs"
  tags   = local.common_tags
}
```

`local.name_prefix` を変えれば全リソースの名前が一括で変わる。

---

## variable との違い

| | `variable` | `locals` |
|---|---|---|
| 外部から値を渡せる | ✅ | ❌ |
| 式・関数が使える | default のみ | ✅ |
| 用途 | 環境差分などの外部入力 | コード内の共通値・計算結果 |

`var.env` のように外から切り替えたい値は `variable`、それを組み合わせて作る `name_prefix` のような内部計算は `locals` に置く、という使い分け。
