# 環境変数による変数の渡し方

`TF_VAR_` プレフィックスを持つ環境変数は、Terraform の入力変数として自動的に読み込まれる。

---

## 基本的な使い方

```bash
# TF_VAR_変数名 で設定する
export TF_VAR_env="production"
export TF_VAR_instance_type="t3.large"

terraform apply
# -var オプションなしで変数が渡される
```

対応する変数定義：

```hcl
variable "env" {
  type = string
}

variable "instance_type" {
  type = string
}
```

---

## CI/CD での活用

GitHub Actions などで Secrets を環境変数として渡す。

```yaml
- name: Terraform Apply
  env:
    TF_VAR_db_password: ${{ secrets.DB_PASSWORD }}
    TF_VAR_api_key: ${{ secrets.API_KEY }}
  run: terraform apply -auto-approve
```

`-var-file` を使わずに機密情報を渡せるため、シークレットを tfvars ファイルに書くリスクを避けられる。

---

## 型の扱い

複雑な型（list, map）は JSON 形式で渡す。

```bash
export TF_VAR_availability_zones='["ap-northeast-1a","ap-northeast-1c"]'
export TF_VAR_tags='{"Env":"prod","Team":"platform"}'
```

---

## 優先順位

環境変数は `terraform.tfvars` や `*.auto.tfvars` より**低い優先度**になる。同じ変数が複数の方法で渡された場合、`-var` > `*.auto.tfvars` > `terraform.tfvars` > 環境変数 > `default` の順で適用される。
