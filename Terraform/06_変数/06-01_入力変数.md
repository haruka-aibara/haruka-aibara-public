# 入力変数

モジュールや設定への入力値を外部から受け取るための仕組み。ハードコードを避け、再利用性を高める。

---

## variable ブロック

```hcl
variable "instance_type" {
  type        = string
  default     = "t3.micro"
  description = "EC2 インスタンスタイプ"
}

variable "env" {
  type = string
  # default なし = 必須変数
}
```

---

## 参照方法

```hcl
resource "aws_instance" "web" {
  instance_type = var.instance_type
  ...
}
```

---

## 変数の渡し方

**コマンドライン**
```bash
terraform apply -var="env=production" -var="instance_type=t3.large"
```

**変数定義ファイル**
```bash
terraform apply -var-file="production.tfvars"
```

**環境変数**
```bash
export TF_VAR_env="production"
terraform apply
```
`TF_VAR_` プレフィックスをつけた環境変数は自動的に変数として読み込まれる。

---

## 優先順位（高い順）

1. `-var` コマンドライン引数
2. `-var-file` で指定したファイル
3. `*.auto.tfvars`（自動読み込み）
4. `terraform.tfvars`（自動読み込み）
5. 環境変数 `TF_VAR_*`
6. `default` 値
