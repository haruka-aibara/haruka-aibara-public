# State ロック

複数人が同時に `terraform apply` を実行して State が競合するのを防ぐ仕組み。

---

## ロックの仕組み

`terraform plan` / `apply` / `destroy` 実行中は State がロックされる。他のユーザーが同時に実行しようとすると以下のエラーになる。

```
Error: Error acquiring the state lock

Error message: ConditionalCheckFailedException: ...
Lock Info:
  ID:        3f1234ab-...
  Path:      my-bucket/prod/terraform.tfstate
  Operation: apply
  Who:       alice@example.com
  Version:   1.5.0
  Created:   2024-01-15 10:30:00 +0000 UTC
```

---

## DynamoDB によるロック（S3 バックエンド）

```hcl
terraform {
  backend "s3" {
    bucket         = "my-terraform-state"
    key            = "prod/terraform.tfstate"
    region         = "ap-northeast-1"
    dynamodb_table = "terraform-lock"  # ロック用テーブル
  }
}
```

DynamoDB テーブルの作成（パーティションキーは `LockID`、型は `String`）：

```hcl
resource "aws_dynamodb_table" "tflock" {
  name         = "terraform-lock"
  billing_mode = "PAY_PER_REQUEST"
  hash_key     = "LockID"

  attribute {
    name = "LockID"
    type = "S"
  }
}
```

---

## ロックの強制解除

プロセスが異常終了してロックが残った場合は強制解除できる。

```bash
terraform force-unlock LOCK_ID
```

`LOCK_ID` はエラーメッセージに表示される。本当にプロセスが終了したことを確認してから実行すること。
