# リモート状態（Remote State）

State ファイルをクラウドストレージで管理する仕組み。チーム開発では必須。

---

## なぜリモートが必要か

ローカルの `terraform.tfstate` をそのまま使うと：

- チームメンバー間で State が共有できない
- 誤って State を削除・破損するリスクがある
- Git に State をコミットすると機密情報が漏洩する

---

## S3 バックエンド（AWS で最も一般的）

```hcl
terraform {
  backend "s3" {
    bucket         = "my-terraform-state"
    key            = "prod/terraform.tfstate"
    region         = "ap-northeast-1"
    encrypt        = true          # サーバーサイド暗号化
    dynamodb_table = "terraform-lock"  # State ロック用
  }
}
```

設定変更後は `terraform init -reconfigure` で再初期化。

---

## GCS バックエンド（GCP）

```hcl
terraform {
  backend "gcs" {
    bucket = "my-terraform-state"
    prefix = "prod"
  }
}
```

---

## data.terraform_remote_state

別の Terraform モジュールの出力値を参照する。

```hcl
# network モジュールの State から vpc_id を取得
data "terraform_remote_state" "network" {
  backend = "s3"
  config = {
    bucket = "my-terraform-state"
    key    = "network/terraform.tfstate"
    region = "ap-northeast-1"
  }
}

resource "aws_subnet" "app" {
  vpc_id = data.terraform_remote_state.network.outputs.vpc_id
}
```

モジュール間の依存関係を密結合にしすぎないよう、参照は最小限にする。

---

## S3 バケットの設定

```hcl
# State 保存用 S3 バケット
resource "aws_s3_bucket" "tfstate" {
  bucket = "my-terraform-state"
}

resource "aws_s3_bucket_versioning" "tfstate" {
  bucket = aws_s3_bucket.tfstate.id
  versioning_configuration { status = "Enabled" }
}

resource "aws_s3_bucket_server_side_encryption_configuration" "tfstate" {
  bucket = aws_s3_bucket.tfstate.id
  rule {
    apply_server_side_encryption_by_default { sse_algorithm = "AES256" }
  }
}
```
