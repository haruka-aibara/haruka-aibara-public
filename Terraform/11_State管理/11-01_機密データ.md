# State と機密データ

Terraform の State ファイルには、リソースの全属性が平文で記録される。DB のパスワードや秘密鍵など、機密情報が含まれることがある。

---

## 機密情報が State に記録される例

```hcl
resource "aws_db_instance" "main" {
  username = "admin"
  password = var.db_password  # State に平文で記録される
  ...
}

resource "aws_iam_access_key" "ci" {
  user = aws_iam_user.ci.name
  # secret が State に記録される
}
```

`output` に `sensitive = true` を設定しても State には平文で保存される。

---

## 対策

### 1. State ファイルを暗号化して保存する

```hcl
# S3 バックエンドでの暗号化
terraform {
  backend "s3" {
    bucket  = "my-tfstate"
    key     = "prod/terraform.tfstate"
    region  = "ap-northeast-1"
    encrypt = true  # SSE-S3 または SSE-KMS
  }
}
```

### 2. State へのアクセスを IAM で制限する

State バケットへの読み書きは CI/CD ロールと管理者のみに絞る。

### 3. 機密情報を State に入れない設計にする

シークレットの管理を Terraform 外で行い、Terraform からは参照するだけにする。

```hcl
# Secrets Manager でシークレットを作成し（Terraform 外）、
# data で参照するだけにする
data "aws_secretsmanager_secret_version" "db" {
  secret_id = "prod/myapp/db"
}

resource "aws_db_instance" "main" {
  password = jsondecode(data.aws_secretsmanager_secret_version.db.secret_string)["password"]
}
```

---

## State ファイルへのアクセスログ

S3 バックエンドでは CloudTrail によるアクセスログを有効にしておくと、誰が State を読み書きしたかを追跡できる。
