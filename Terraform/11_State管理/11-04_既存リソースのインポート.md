# 既存リソースのインポート

手動で作成したリソースや、他のツールで管理しているリソースを Terraform の管理下に取り込む。

---

## import ブロック（v1.5+ 推奨）

`import` ブロックで宣言的にインポートできる。

```hcl
# import ブロックで取り込む対象を宣言
import {
  to = aws_instance.web
  id = "i-0abc123def456789"
}

# 対応するリソース定義も必要
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"
}
```

`terraform plan` で差分確認後、`terraform apply` でインポートが完了する。

---

## terraform import コマンド（旧来の方法）

```bash
terraform import aws_instance.web i-0abc123def456789
```

State にリソースを取り込むだけなので、その後 `.tf` ファイルに対応するリソース定義を書く必要がある。

---

## -generate-config-out オプション（v1.5+）

インポート時に `.tf` の雛形を自動生成できる。

```bash
terraform plan -generate-config-out=generated.tf
```

生成されたファイルを確認・修正してから使う。すべての属性が正確に生成されるわけではないため、プロバイダーのドキュメントと照合する。

---

## インポートの注意点

- リソース ID の形式はリソースタイプごとに異なる（ドキュメントの `Import` セクションを参照）
- インポート後に `terraform plan` を実行して差分がないことを確認する
- 差分がある場合は `.tf` の定義を修正するか `ignore_changes` で無視する
