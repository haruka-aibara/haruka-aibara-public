# State のバージョン管理

State ファイルのバージョン管理で、誤操作時のロールバックや変更履歴の追跡を可能にする。

---

## S3 バケットのバージョニング

```hcl
resource "aws_s3_bucket_versioning" "tfstate" {
  bucket = aws_s3_bucket.tfstate.id
  versioning_configuration {
    status = "Enabled"
  }
}
```

バージョニングを有効にすると、State が更新されるたびに前のバージョンが保持される。

---

## 過去バージョンへのロールバック

```bash
# 過去バージョンの一覧を確認（AWS CLI）
aws s3api list-object-versions \
  --bucket my-tfstate \
  --prefix prod/terraform.tfstate

# 特定バージョンをダウンロード
aws s3api get-object \
  --bucket my-tfstate \
  --key prod/terraform.tfstate \
  --version-id VERSION_ID \
  backup.tfstate

# State を戻す（慎重に）
terraform state push backup.tfstate
```

---

## State のバックアップ

手動操作前に必ずバックアップを取る習慣をつける。

```bash
# 現在の State をローカルに保存
terraform state pull > backup-$(date +%Y%m%d-%H%M%S).tfstate
```

---

## terraform.tfstate を Git にコミットしない

`.gitignore` に追加する。State には機密情報が含まれる可能性があり、履歴に残ると削除しにくい。

```gitignore
*.tfstate
*.tfstate.backup
.terraform/
```

State のバージョン管理は S3 のバージョニング機能で行い、Git では管理しない。
