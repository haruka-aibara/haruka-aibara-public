# terraform init

`.tf` ファイルを書いたら最初に実行するコマンド。プロバイダープラグインのダウンロードとバックエンドの初期化を行う。

---

## 実行内容

```bash
terraform init
```

1. `required_providers` に書かれたプロバイダープラグインを `.terraform/` にダウンロード
2. バックエンド（S3 等）の設定を初期化
3. `terraform.lock.hcl` を生成・更新

---

## .terraform/ ディレクトリ

`init` で作られるプロバイダーバイナリの保存場所。`.gitignore` に追加するのが基本。

```gitignore
.terraform/
*.tfstate
*.tfstate.backup
```

---

## terraform.lock.hcl

プロバイダーのバージョンを固定するロックファイル。**Git にコミットする**ことで、チーム全員が同じバージョンを使うことを保証できる。

```hcl
provider "registry.terraform.io/hashicorp/aws" {
  version     = "5.31.0"
  constraints = "~> 5.0"
  hashes = ["h1:..."]
}
```

---

## よく使うオプション

```bash
# バックエンドの設定を変更して再初期化
terraform init -reconfigure

# プロバイダーをアップグレード（lock.hcl も更新される）
terraform init -upgrade

# バックエンド設定を外部から渡す（CI/CD での使用例）
terraform init -backend-config="bucket=my-tfstate-bucket"
```
