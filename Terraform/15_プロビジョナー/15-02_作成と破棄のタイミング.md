# 作成時・破棄時プロビジョナー

「インスタンスを作ったときだけスクリプトを実行したい」「削除するときに外部サービスから登録解除したい」——プロビジョナーの実行タイミングは `when` で制御できる。

---

## 作成時プロビジョナー（デフォルト）

`when = create`（省略可）。リソースが新規作成されたときだけ実行される。

```hcl
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"

  provisioner "local-exec" {
    command = "echo ${self.public_ip} >> ./ip_list.txt"
    # when = create  ← デフォルトなので省略可
  }
}
```

リソースを更新しても再実行されない（replace の場合は新規作成なので実行される）。

---

## 破棄時プロビジョナー（destroy）

`when = destroy` を指定すると、リソース削除時に実行される。

```hcl
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"

  provisioner "local-exec" {
    when    = destroy
    command = "echo 'Deregistering ${self.id} from load balancer...'"
  }
}
```

---

## on_failure

プロビジョナーが失敗したときの挙動を制御する。

```hcl
provisioner "local-exec" {
  command    = "./register.sh ${self.id}"
  on_failure = continue  # 失敗しても apply を続行
  # on_failure = fail    # デフォルト：失敗したら apply も失敗
}
```

`continue` にすると失敗を無視するため、冪等でない処理で使う場合は注意。

---
