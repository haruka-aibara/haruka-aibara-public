# ルートモジュールとサブモジュール

---

## モジュールとは

リソースのまとまりをパッケージ化して再利用するための仕組み。VPC の構成、ECS クラスターなど、繰り返し使うパターンをモジュール化することでコードの重複を防ぐ。

---

## ルートモジュール

`terraform apply` を実行するディレクトリ（= 作業ディレクトリ）のこと。すべての Terraform 設定は何らかのルートモジュールの中にある。

---

## サブモジュール（子モジュール）

ルートモジュールから呼び出されるモジュール。`module` ブロックで呼び出す。

```hcl
# ローカルモジュールを呼び出す
module "network" {
  source = "./modules/network"

  vpc_cidr = "10.0.0.0/16"
  env      = var.env
}

# レジストリのモジュールを呼び出す
module "s3_bucket" {
  source  = "terraform-aws-modules/s3-bucket/aws"
  version = "~> 4.0"

  bucket = "my-bucket"
}
```

---

## モジュールのディレクトリ構成

```
project/
├── main.tf          # ルートモジュール（module ブロックで呼び出し）
├── variables.tf
├── outputs.tf
└── modules/
    ├── network/     # サブモジュール
    │   ├── main.tf
    │   ├── variables.tf
    │   └── outputs.tf
    └── app/
        ├── main.tf
        ├── variables.tf
        └── outputs.tf
```

---

## モジュールの出力を参照する

```hcl
# modules/network/outputs.tf
output "vpc_id" {
  value = aws_vpc.main.id
}

# ルートモジュール
resource "aws_subnet" "app" {
  vpc_id = module.network.vpc_id
}
```
