# terraform state mv

State 内のリソースの名前・アドレスを変更するコマンド。リファクタリング時に使う。

---

## 基本的な使い方

```bash
terraform state mv SOURCE DESTINATION
```

---

## 使いどころ

### リソース名のリネーム

```hcl
# 変更前
resource "aws_instance" "web" { ... }

# 変更後
resource "aws_instance" "web_server" { ... }
```

`.tf` を書き換えただけだと plan で「削除 + 新規作成」になる。`mv` を先に実行することで replace を回避できる。

```bash
terraform state mv aws_instance.web aws_instance.web_server
```

### モジュールへの移動

```bash
# トップレベルのリソースをモジュール内に移動
terraform state mv aws_instance.web module.app.aws_instance.web
```

---

## moved ブロック（v1.1+ 推奨）

コマンドを手動実行しなくても、`.tf` にリファクタリングの意図を宣言できる。

```hcl
moved {
  from = aws_instance.web
  to   = aws_instance.web_server
}
```

`terraform apply` を実行すると自動的に State 上で移動が行われる。チームで共有しやすく、Git で履歴も残る。

---

## 注意点

- `mv` は State のみ変更する。実際のリソースに影響はない
- 操作前にバックアップを取ること
- `moved` ブロックが使える場合はそちらを優先する
