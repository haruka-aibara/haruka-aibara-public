# terraform state show

apply 後に「このインスタンスの IP は何番だっけ」「ARN を確認したい」というとき、コンソールを開かなくても `terraform state show` で State に記録された属性を一覧できる。`output` に定義していない属性も含めて確認できるのが便利。

---

## 基本的な使い方

```bash
terraform state show aws_instance.web
```

出力例：

```hcl
# aws_instance.web:
resource "aws_instance" "web" {
    ami           = "ami-0c55b159cbfafe1f0"
    arn           = "arn:aws:ec2:ap-northeast-1:123456789:instance/i-0abc123"
    id            = "i-0abc123def456789"
    instance_type = "t3.micro"
    private_ip    = "10.0.1.100"
    public_ip     = "54.123.45.67"
    ...
}
```

---

## for_each / count の場合

```bash
# for_each（キーを指定）
terraform state show 'aws_s3_bucket.example["logs"]'

# count（インデックスを指定）
terraform state show 'aws_instance.web[0]'
```

特殊文字が含まれる場合はシングルクォートで囲む。

---

## よく使う場面

- apply 後のリソース ID・ARN・IP を素早く確認する
- `output` に書いていない属性を確認する
- `state rm` や `state mv` の前に対象リソースの現状を把握する
