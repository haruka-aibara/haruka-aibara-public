# terraform state list

「今 Terraform で何を管理しているのか全体像を把握したい」「`state rm` や `state show` で使うリソース名がわからない」——そういうときに使うのが `terraform state list`。State に登録されているリソースの一覧を出力する。

---

## 基本的な使い方

```bash
terraform state list
```

出力例：

```
aws_instance.web
aws_security_group.web
aws_s3_bucket.logs
aws_s3_bucket.example["backups"]
aws_s3_bucket.example["logs"]
module.network.aws_vpc.main
module.network.aws_subnet.public[0]
module.network.aws_subnet.public[1]
```

---

## フィルタリング

```bash
# 特定のリソースタイプのみ
terraform state list aws_instance.*

# モジュール内のリソースのみ
terraform state list module.network
```

---

## よく使う場面

- `state show <リソース名>` で詳細を調べる前にリソース名を確認する
- モジュールリファクタリング後、意図したアドレスになっているか検証する
- 意図せず State に残っているリソースがないか確認する
