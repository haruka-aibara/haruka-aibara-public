# リソースライフサイクル

Terraform がリソースを作成・更新・削除する際のデフォルト動作と、`lifecycle` ブロックによるカスタマイズ。

---

## デフォルトの動作

```
1. terraform plan  → 差分を計算
2. terraform apply → 差分を適用
   - 新規リソース: Create
   - 変更可能な属性: Update（in-place）
   - 変更不可な属性: Delete → Create（replace）
3. terraform destroy → 全リソースを Delete
```

---

## replace が発生する例

EC2 インスタンスの `ami` を変更するとインスタンスの再作成が必要になる。

```hcl
resource "aws_instance" "web" {
  ami           = "ami-new"  # 変更すると replace 発生
  instance_type = "t3.micro"
}
```

`plan` で `forces replacement` と表示されたら注意。

---

## lifecycle ブロック

```hcl
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"

  lifecycle {
    create_before_destroy = true  # 新しいものを作ってから古いものを削除
    prevent_destroy       = true  # 誤削除を防ぐ（destroy しようとするとエラー）
    ignore_changes        = [ami] # 指定属性の変更を無視
  }
}
```

詳細は `05-03_メタ引数/lifecycle` を参照。
