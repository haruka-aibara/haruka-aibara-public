# リソースの挙動

---

## resource ブロック

インフラのコンポーネントを定義する基本ブロック。

```hcl
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"

  tags = {
    Name = "web-server"
  }
}
```

`resource "リソースタイプ" "ローカル名" { ... }` の形式。`aws_instance.web` のように他のリソースから参照できる。

---

## apply 時の CRUD 動作

| 操作 | タイミング |
|---|---|
| **Create** | State にないリソースが .tf に存在する |
| **Read** | `plan` 実行時、現在の状態を API で取得 |
| **Update** | State と .tf の差分があり in-place 更新が可能 |
| **Delete** | .tf から削除されたリソースが State に残っている |

---

## Replace（再作成）

変更が in-place 更新できない場合（例: EC2 の AMI 変更）はリソースの削除・再作成が発生する。`plan` の出力で `-/+` と表示される。

```
# aws_instance.web must be replaced
-/+  resource "aws_instance" "web" {
       ami = "ami-old" -> "ami-new"  # forces replacement
     }
```

意図せず Replace が発生しないよう `plan` を必ず確認すること。

---

## リソースへの参照

同一モジュール内のリソースは `タイプ.名前.属性` で参照できる。

```hcl
resource "aws_eip" "web" {
  instance = aws_instance.web.id  # 上記リソースの id を参照
}
```

Terraform は依存関係を自動解決し、参照元リソースを先に作成する。
