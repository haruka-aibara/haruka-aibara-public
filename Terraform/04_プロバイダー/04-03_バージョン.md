# プロバイダーのバージョン管理

---

## バージョン制約の書き方

```hcl
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
  required_version = ">= 1.5"
}
```

---

## 制約演算子

| 演算子 | 意味 | 例 |
|---|---|---|
| `= 5.31.0` | 完全固定 | `5.31.0` のみ |
| `>= 5.0` | 以上 | `5.0.0` 以上 |
| `~> 5.0` | パッチのみ更新許可 | `5.0.x` のみ（`5.1` は NG） |
| `~> 5` | マイナーまで更新許可 | `5.x.x` のみ（`6` は NG） |
| `>= 5.0, < 6.0` | 範囲指定 | `5.x` のみ |

実務では `~>` を使うことが多い。メジャーバージョン跨ぎの破壊的変更を防ぎつつ、パッチアップデートを許容できる。

---

## terraform.lock.hcl との関係

`required_providers` の制約を満たすバージョンの中から実際にダウンロードしたバージョンが `terraform.lock.hcl` に記録される。

```hcl
provider "registry.terraform.io/hashicorp/aws" {
  version     = "5.31.0"
  constraints = "~> 5.0"
  hashes = ["h1:..."]
}
```

`terraform init -upgrade` で制約の範囲内の最新バージョンに更新できる。

---

## バージョンアップ時の注意

プロバイダーのメジャーバージョンアップは破壊的変更を含むことがある。アップグレード前にプロバイダーの Changelog と移行ガイドを確認すること。
