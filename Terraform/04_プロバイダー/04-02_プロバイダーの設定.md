# プロバイダーの設定

---

## 基本的な書き方

`terraform` ブロックで使用するプロバイダーを宣言し、`provider` ブロックで設定する。

```hcl
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

provider "aws" {
  region = "ap-northeast-1"
}
```

---

## 認証情報の渡し方（AWS の例）

**環境変数（推奨）**

```bash
export AWS_ACCESS_KEY_ID="AKIA..."
export AWS_SECRET_ACCESS_KEY="..."
export AWS_REGION="ap-northeast-1"
```

`provider` ブロックに直接書かない。コードに認証情報を含めると Git に入ってしまうリスクがある。

**プロファイルを使う**

```hcl
provider "aws" {
  region  = "ap-northeast-1"
  profile = "my-profile"
}
```

**IAM ロール（EC2 / ECS / Lambda 上）**

インスタンスプロファイルが自動的に使われるため設定不要。CI/CD でも OIDC + IAM ロールの組み合わせが推奨。

---

## 複数リージョンへのデプロイ

`alias` を使って同一プロバイダーを複数設定できる。

```hcl
provider "aws" {
  region = "ap-northeast-1"
}

provider "aws" {
  alias  = "us_east"
  region = "us-east-1"
}

resource "aws_s3_bucket" "us_bucket" {
  provider = aws.us_east
  bucket   = "my-us-bucket"
}
```
