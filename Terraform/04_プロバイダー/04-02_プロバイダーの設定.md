# プロバイダーの設定

Terraform でクラウドリソースを作るには、まず「どのクラウドを使うか」「どう認証するか」を設定する必要がある。ここでミスると apply すらできないし、認証情報をコードに書くと即セキュリティ事故になる。

---

## 基本的な書き方

```hcl
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

provider "aws" {
  region = "ap-northeast-1"
}
```

---

## 認証情報の渡し方（AWS）

**環境変数が基本。コードには絶対書かない。**

```bash
export AWS_ACCESS_KEY_ID="AKIA..."
export AWS_SECRET_ACCESS_KEY="..."
export AWS_REGION="ap-northeast-1"
```

`provider` ブロックに直接書くと Git にコミットしてしまうリスクがある。それだけは避ける。

**AWS プロファイルを使う場合**

```hcl
provider "aws" {
  region  = "ap-northeast-1"
  profile = "my-profile"   # ~/.aws/credentials のプロファイル名
}
```

**EC2・ECS・Lambda 上で実行する場合**

インスタンスプロファイルや実行ロールが自動で使われるため、認証設定は不要。CI/CD でも OIDC + IAM ロールの組み合わせが現在の主流。

---

## 複数リージョンにデプロイしたい場合

CloudFront の証明書は us-east-1 に置く必要があるなど、複数リージョンを同時に扱いたい場面がある。`alias` でプロバイダーを複数定義する。

```hcl
provider "aws" {
  region = "ap-northeast-1"   # デフォルト
}

provider "aws" {
  alias  = "us_east"
  region = "us-east-1"
}

resource "aws_acm_certificate" "cert" {
  provider = aws.us_east   # us-east-1 に作る
  domain_name = "example.com"
}
```
