# HCL（HashiCorp Configuration Language）とは

HashiCorp が開発した構成記述言語。Terraform の `.tf` ファイルで使われる。JSON よりも人間が読み書きしやすく設計されており、コメントや式の評価もサポートしている。

---

## JSON との比較

同じ設定を HCL と JSON で書いた場合：

**HCL**
```hcl
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"

  tags = {
    Name = "web-server"
  }
}
```

**JSON 相当**
```json
{
  "resource": {
    "aws_instance": {
      "web": {
        "ami": "ami-0c55b159cbfafe1f0",
        "instance_type": "t3.micro",
        "tags": { "Name": "web-server" }
      }
    }
  }
}
```

HCL は引用符やネストが少なく、コメントも書けるため実務ではほぼ HCL のみ使われる。

---

## ファイル構成の慣習

複数の `.tf` ファイルを同一ディレクトリに置ける（Terraform がまとめて読む）。役割ごとに分割するのが慣習。

```
main.tf        # リソース定義
variables.tf   # 入力変数
outputs.tf     # 出力値
providers.tf   # プロバイダー設定
```
