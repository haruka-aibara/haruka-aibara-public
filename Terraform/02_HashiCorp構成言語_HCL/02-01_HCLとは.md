# HCL（HashiCorp Configuration Language）とは

Terraform のコードを書くときに使う言語。JSON でも書けるが、現実的には誰も JSON では書かない。HCL は人間が読み書きするために設計されているので、コメントが書けて、ネストが浅くて済む。

---

## JSON との比較

同じ設定を書いた場合：

**HCL**
```hcl
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"

  tags = {
    Name = "web-server"
  }
}
```

**JSON 相当**
```json
{
  "resource": {
    "aws_instance": {
      "web": {
        "ami": "ami-0c55b159cbfafe1f0",
        "instance_type": "t3.micro",
        "tags": { "Name": "web-server" }
      }
    }
  }
}
```

引用符・中括弧のネストが多く、コメントも書けない JSON はメンテが辛い。HCL 一択。

---

## ファイルの分け方

同一ディレクトリに複数の `.tf` ファイルを置ける。Terraform がまとめて読む。役割で分けるのが慣習。

```
main.tf        # リソース定義
variables.tf   # 入力変数
outputs.tf     # 出力値
providers.tf   # プロバイダー設定
```

すべてを `main.tf` に書いても動くが、大きくなると探しにくい。最初から分けておくと後が楽。
