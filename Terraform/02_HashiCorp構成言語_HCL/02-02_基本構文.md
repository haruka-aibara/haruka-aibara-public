# HCL 基本構文

Terraform のコードを読み書きするときに押さえておく最低限の構文。これを知っていればほとんどのコードは読める。

---

## ブロック

HCL の基本単位。`ブロックタイプ "ラベル" { ... }` の形式。

```hcl
resource "aws_s3_bucket" "example" {
  bucket = "my-bucket-name"
}
```

- `resource` → ブロックタイプ
- `"aws_s3_bucket"` → リソースタイプ
- `"example"` → このファイル内での名前（他のブロックから参照するときに使う）

---

## 値の型

```hcl
name    = "example"                               # 文字列
count   = 3                                       # 数値
enabled = true                                    # ブール
azs     = ["ap-northeast-1a", "ap-northeast-1c"]  # リスト
tags    = { Env = "prod", Team = "platform" }     # マップ
```

---

## リソース間の参照

`タイプ.名前.属性` で他のリソースの値を参照できる。

```hcl
resource "aws_subnet" "app" {
  vpc_id = aws_vpc.main.id  # aws_vpc リソース "main" の id を参照
}
```

この参照があると Terraform が依存関係を自動解析して作成順序を決める。`depends_on` を手で書く必要がない。

変数: `var.変数名` / ローカル値: `local.名前` / データソース: `data.タイプ.名前.属性`

---

## 式と関数

```hcl
# 文字列補間
name = "app-${var.env}-server"

# 組み込み関数
upper_name = upper(var.name)

# 条件式
instance_type = var.env == "production" ? "t3.large" : "t3.micro"
```

---

## コメント

```hcl
# 一行コメント
// 一行コメント（どちらでも可）

/*
  複数行コメント
*/
```
