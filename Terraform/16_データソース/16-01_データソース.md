# データソース（data）

Terraform は `resource` ブロックで作ったものしか「知らない」。でも現実には、手動で作った VPC・別チームが管理している Route53 ホストゾーン・AWS が提供している最新の AMI など、**自分の管理外のリソースを参照したい場面がたくさんある**。

それを解決するのが `data` ブロック。作成や変更はせず、読み取るだけ。

---

## 使い方

```hcl
data "aws_vpc" "default" {
  default = true
}

data "aws_ami" "amazon_linux" {
  most_recent = true
  owners      = ["amazon"]

  filter {
    name   = "name"
    values = ["al2023-ami-*-x86_64"]
  }
}
```

参照するには `data.タイプ.名前.属性`：

```hcl
resource "aws_instance" "web" {
  ami       = data.aws_ami.amazon_linux.id   # 最新 AMI ID を自動取得
  subnet_id = data.aws_vpc.default.id
}
```

AMI ID をハードコードすると古くなる。`data` で取得すれば常に最新を参照できる。

---

## よく使う場面

### 別チームが管理しているリソースを参照したい

```hcl
# ネットワークチームが管理している VPC のサブネットを使いたい
data "aws_subnet" "private" {
  filter {
    name   = "tag:Name"
    values = ["shared-private-subnet-1a"]
  }
}

resource "aws_ecs_service" "app" {
  network_configuration {
    subnets = [data.aws_subnet.private.id]
  }
}
```

### 自分の Terraform で作ったわけではない既存リソースを使いたい

```hcl
# 手動で作った Route53 ホストゾーンに DNS レコードを追加
data "aws_route53_zone" "main" {
  name = "example.com."
}

resource "aws_route53_record" "app" {
  zone_id = data.aws_route53_zone.main.zone_id
  name    = "app.example.com"
  type    = "A"
}
```

### AWS 側が管理している情報を取得したい

```hcl
# 実行中の AWS アカウント ID やリージョンを動的に取得
data "aws_caller_identity" "current" {}
data "aws_region" "current" {}

resource "aws_s3_bucket" "logs" {
  bucket = "logs-${data.aws_caller_identity.current.account_id}-${data.aws_region.current.name}"
}
```

---

## resource との違い

| | `resource` | `data` |
|---|---|---|
| 何をするか | リソースを作る・管理する | 既存の情報を読むだけ |
| apply の影響 | 作成・更新・削除が発生する | 何も変わらない |
| 典型的な用途 | 自分のチームが所有するリソース | 他チーム管理・AWS 管理・手動作成のリソース |
