# 機密出力（sensitive outputs）

DB パスワードや API キーを `output` に出すとターミナルに丸見えになる。CI のログに残ってしまうケースもある。`sensitive = true` を設定することで、表示をマスクできる。

---

## sensitive = true の設定

```hcl
output "db_password" {
  value     = aws_db_instance.main.password
  sensitive = true
}
```

apply やログ確認のときに値が隠れる：

```
Outputs:

db_password = (sensitive value)
```

---

## `terraform output -json` では値が見える

`sensitive = true` はあくまでターミナル表示のマスクであって、JSON 出力では値が含まれる。

```bash
terraform output -json db_password
# → {"sensitive": true, "value": "実際のパスワード", ...}
```

CI のログに `terraform output -json` の結果を流し込む場合は注意。ログマスクの設定を別途行う。

---

## State ファイルには平文で記録される

`sensitive = true` に関わらず、**State ファイルには平文で保存される**。出力値だけでなく、State 自体を守る必要がある。

- S3 バックエンドでは `encrypt = true` を設定する
- バケットは IAM でアクセスを制限する
- `terraform.tfstate` は絶対に Git にコミットしない
