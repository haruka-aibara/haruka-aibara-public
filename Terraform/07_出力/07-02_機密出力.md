# 機密出力（sensitive outputs）

パスワードや API キーなどの機密情報を出力値として扱う場合、`sensitive = true` を設定する。

---

## sensitive = true

```hcl
output "db_password" {
  value     = aws_db_instance.main.password
  sensitive = true
}
```

`terraform apply` や `terraform output` の結果で `(sensitive value)` と表示され、値がマスクされる。

```
Outputs:

db_password = (sensitive value)
```

---

## JSON で取得する場合

`terraform output -json` では sensitive な値も含めて出力される（値は見える）ため注意。

```bash
# JSON 出力には sensitive 値も含まれる
terraform output -json db_password
```

CI/CD パイプラインで `terraform output -json` を使う場合はログに機密情報が残らないよう注意する。

---

## State ファイルへの保存

`sensitive = true` の設定に関わらず、出力値は `terraform.tfstate` にプレーンテキストで保存される。State ファイル自体を安全に管理することが重要。

- S3 バックエンドでは SSE（サーバーサイド暗号化）を有効にする
- アクセスは IAM で制限する
- `terraform.tfstate` を Git にコミットしない
