# 事前条件（precondition）

output ブロックに `precondition` を設定して、出力値を返す前に条件を検証できる。条件を満たさない場合は apply がエラーになる。

---

## 基本的な書き方

```hcl
output "load_balancer_dns" {
  value = aws_lb.main.dns_name

  precondition {
    condition     = aws_lb.main.enable_deletion_protection
    error_message = "本番 LB は削除保護を有効にしてください。"
  }
}
```

---

## resource の precondition / postcondition

output だけでなく、`resource` ブロックの `lifecycle` 内にも `precondition` / `postcondition` を書ける（v1.2+）。

```hcl
resource "aws_instance" "web" {
  ami           = data.aws_ami.app.id
  instance_type = var.instance_type

  lifecycle {
    precondition {
      condition     = data.aws_ami.app.architecture == "x86_64"
      error_message = "AMI は x86_64 アーキテクチャを使ってください。"
    }

    postcondition {
      condition     = self.public_ip != ""
      error_message = "インスタンスにパブリック IP が割り当てられませんでした。"
    }
  }
}
```

- `precondition`: apply 前に評価。前提条件のチェック
- `postcondition`: apply 後に評価。`self` でリソースの属性を参照できる

---

## validation との使い分け

| | variable validation | precondition / postcondition |
|---|---|---|
| 評価タイミング | plan 前（変数入力時） | plan / apply 時 |
| 参照できる値 | `var.変数名` のみ | リソース属性・data ソースも参照可能 |
| 用途 | 入力値の検証 | インフラの状態・設定の検証 |
