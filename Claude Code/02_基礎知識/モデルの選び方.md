# モデルの選び方（Opus / Sonnet / Haiku）

## こういうときに迷う

「なんとなく Sonnet を使っているけど、Opus の方がいいのかわからない」「Haiku でいいケースに Opus を使ってお金を無駄にしている気がする」。

Claude Code には 3 種類のモデルがあり、タスクに合わせて使い分けると品質とコストを最適化できる。

---

## 3 モデルの役割分担

### Claude Opus — 深く考えるとき

複雑な推論・大規模なアーキテクチャ設計・難しいバグの特定に使う。時間はかかるがアウトプットの質が高い。

**向いているタスク：**
- 大きなリファクタリングの設計
- 複数ファイルにまたがるバグの原因特定
- 初めて触るコードベースの構造理解
- セキュリティ・パフォーマンスの深い分析

### Claude Sonnet — 普段使いのバランス型

速度と質のバランスが良く、日常的なコーディング作業のデフォルト。ほとんどの作業はこれで十分。

**向いているタスク：**
- 機能の実装・修正
- テストの追加
- コードレビュー
- ドキュメント作成

### Claude Haiku — 速さとコストを重視するとき

軽量・高速・安価。単純な作業や、サブエージェントに大量処理させる場合に使う。

**向いているタスク：**
- 簡単な質問・検索
- コミットメッセージの生成
- サブエージェントでの並列調査
- CI でのちょっとしたチェック

## モデルの切り替え方

### セッション内で切り替える

```
/model
```

モデル選択メニューが開く。

### 起動時に指定する

```bash
claude --model claude-opus-4-6
claude --model claude-sonnet-4-6
claude --model claude-haiku-4-5
```

### Headless Mode（`-p`）で指定する

```bash
claude -p "このコードを分析して" --model claude-haiku-4-5
```

## プランとデフォルトモデルの関係

| プラン | デフォルトモデル |
|--------|----------------|
| Claude Max、Team Premium | Opus 4.6 |
| Claude Pro、Team Standard | Sonnet 4.6 |
| API（従量課金） | 指定したモデルをフルアクセス |

---

## Fast Mode（Opus 限定）

Opus 4.6 専用で **2.5 倍高速** になるモード。ただしコストも上がる。セッション中に `/fast` でトグルできる。

急いでいる場面で Opus の質が欲しいとき向け。サブスクリプションプランでは extra usage 扱い（プランの使用枠に含まれない）。
