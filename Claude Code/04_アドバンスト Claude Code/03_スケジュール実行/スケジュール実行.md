# スケジュール実行

## こういうときに使いたい

「毎週月曜に依存ライブラリの更新チェックを自動で走らせたい」「日次で特定のログを要約させたい」「定期的なコードレビューを CI に組み込みたい」。

Claude Code の Headless Mode（`-p` フラグ）と cron や GitHub Actions のスケジュール機能を組み合わせると、定期的なタスクを完全自動化できる。

---

## cron で定期実行する

### 基本的な設定

```bash
# crontab -e で編集
# 毎週月曜 9:00 に依存関係チェックを実行
0 9 * * 1 cd /path/to/project && claude --bare -p \
  "package.json を確認して、古くなっているパッケージを列挙してください" \
  --allowedTools "Read" \
  --model claude-haiku-4-5 \
  >> ~/logs/dependency-check.log 2>&1
```

### 環境変数の設定

cron は通常の shell 環境が読み込まれないため、API キーを明示する：

```bash
0 9 * * 1 ANTHROPIC_API_KEY=sk-... cd /path/to/project && claude --bare -p "..."
```

または専用の設定ファイルを用意する：

```bash
# ~/.claude-cron-env
export ANTHROPIC_API_KEY=sk-...
export CLAUDE_CODE_EFFORT_LEVEL=low

# crontab
0 9 * * 1 source ~/.claude-cron-env && cd /path/to/project && claude --bare -p "..."
```

## GitHub Actions でスケジュール実行する

定期タスクを CI に乗せると、ログの管理・通知・PR の自動作成まで一貫して自動化できる。

```yaml
name: Weekly Maintenance
on:
  schedule:
    - cron: '0 9 * * 1'  # 毎週月曜 9:00 UTC

jobs:
  maintenance:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Check dependencies
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          claude --bare -p \
            "package.json を確認して、メジャーバージョンが上がっているパッケージを
             列挙し、breaking changes のリスクを評価してください" \
            --allowedTools "Read,Bash" \
            --model claude-haiku-4-5 \
            --output-format json | jq -r '.result' > report.md

      - name: Create issue if problems found
        # report.md を元に GitHub Issue を作成する処理
```

## スケジュール実行に向いているタスク

| タスク | 頻度の例 |
|--------|---------|
| 依存ライブラリの更新チェック | 週次 |
| セキュリティ脆弱性スキャン | 日次 |
| コードの複雑度レポート | 週次 |
| ドキュメントと実装の乖離チェック | 週次 |
| ログの異常パターン要約 | 日次 |

---

## 注意点

**`--bare` を必ず付ける**
ローカルの CLAUDE.md・hooks が実行環境に存在しない場合でも動作するよう、`--bare` で設定ファイルの読み込みをスキップする。

**軽量モデルを使う**
定期タスクは Haiku で十分なことが多い。コストを抑えるために `--model claude-haiku-4-5` を指定する。

**タイムアウトを設定する**
想定外に処理が長くなったときのために、タイムアウトを設けておく（`timeout` コマンドや GitHub Actions の `timeout-minutes`）。
