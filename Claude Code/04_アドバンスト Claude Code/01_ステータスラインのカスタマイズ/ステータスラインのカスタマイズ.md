# ステータスラインのカスタマイズ

## こういうときに使いたい

「今どのモデルを使っているか毎回確認しに行くのが面倒」「トークン使用量をターミナルに常時表示しておきたい」「セッションの状態をひと目で把握したい」。

Claude Code には 2 種類の状態確認機能がある。

---

## /status — セッションの詳細を一時確認

```
/status
```

タブ形式のダッシュボードが開き、以下を確認できる：

- Claude Code のバージョン
- 使用中のモデル
- アカウント情報
- デイリー・ウィークリーのトークン使用量と上限

作業を始める前に「今日あとどれくらい使えるか」を確認したいときや、接続状態の確認に使う。確認が終わると閉じる一時表示。

---

## /statusline — ターミナル下部に常時表示

```
/statusline
```

ターミナルの下部に常時表示されるバーを設定する。作業中もモデル・コスト・セッション状態などが見える状態になる。

---

## ステータスラインをカスタマイズする

`/statusline` のスクリプトをカスタマイズすることで、表示する情報を変えられる。

設定は `~/.claude/settings.json` の `statusCommand` フィールドにスクリプトを書く：

```json
{
  "statusCommand": "echo '$(date +%H:%M) | model: claude-sonnet'"
}
```

シェルスクリプトを外部ファイルに切り出すこともできる：

```json
{
  "statusCommand": "~/.claude/statusline.sh"
}
```

よく使われる表示例：
- 現在時刻
- 使用中のモデル名
- git ブランチ名
- 今日のトークン消費量
