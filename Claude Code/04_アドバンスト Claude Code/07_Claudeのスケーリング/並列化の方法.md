# Claude Code の並列化

## どんなときに並列化したいか

Claude Code は 1 セッションで 1 つのタスクを順番に処理する。「Terraform の記事を書きながら、Python の記事も書いてほしい」というとき、セッションを並列に走らせることで待ち時間をなくせる。

---

## ケース 1：触るファイルが被らない場合

ディレクトリ・ファイルが完全に分かれているなら、**同じ作業ディレクトリで複数ターミナルを開くだけで OK**。

```
ターミナル 1: claude →「Terraform の記事を書いて」
ターミナル 2: claude →「Python の記事を書いて」
ターミナル 3: claude →「Claude Code の記事を書いて」
```

ただし **コミットだけは被らせない**。複数セッションが同時に `git commit` しようとすると git の内部状態が壊れることがある。各セッションに「ファイルを作るだけでコミットはしないで」と伝えておき、最後に自分でまとめてコミットするのが安全。

---

## ケース 2：同じファイルを複数セッションが触る可能性がある場合

コードの大規模リファクタリングや、共通ファイルを複数の変更が参照するような場合は `git worktree` でセッションを分離する。

```bash
# テーマごとに独立したブランチ・ディレクトリを作る
git worktree add ../repo-feature-a -b feature-a
git worktree add ../repo-bugfix-b -b bugfix-b
```

各ディレクトリで別ターミナルを開いて `claude` を起動すれば、ファイルが完全に分離した状態で並列実行できる。終わったらマージして worktree を削除する。

```bash
git worktree remove ../repo-feature-a
```

---

## ケース 3：大規模な一括変更を Claude に自動で並列処理させたい

全ファイルのリネームやマイグレーションなど「たくさんのファイルを一気に変えたい」ときは `/batch` が使える。複数のエージェントを自動で起動し、それぞれが別の worktree で作業する。

```
/batch 全ての Markdown ファイルの front matter を統一して
```

自分で worktree を作らなくても、Claude Code が内部で分割・並列実行してくれる。

---

## まとめ

| 状況 | 方法 |
|------|------|
| 触るファイルが被らない | 複数ターミナルで `claude` を起動、コミットは自分で |
| 同じファイルを複数セッションが触りうる | `git worktree` でブランチを分離してから起動 |
| 大量ファイルへの一括変更を自動化したい | `/batch` コマンド |
