# 権限モデルと安全な使い方

## Claude Code の権限モデル

Claude Code はすべてのツール操作（ファイル編集、コマンド実行など）に対して権限チェックを行う。勝手にファイルを消したり、本番にデプロイしたりはしない（させない）仕組みになっている。

---

## 3 つの権限モード

起動時に `--allowedTools` フラグや設定ファイルで制御する。

| モード | 動作 |
|-------|------|
| **デフォルト** | 読み取り系は自動許可、書き込み系は毎回確認 |
| **Plan モード** | 読み取りのみ許可。編集やコマンド実行は一切できない |
| **カスタム** | ツール単位で許可/拒否を細かく設定 |

---

## ツールごとの権限設定

`.claude/settings.json` で個別に設定できる。

```json
{
  "permissions": {
    "allow": [
      "Read",
      "Glob",
      "Grep",
      "Bash(npm test)",
      "Bash(npm run lint)"
    ],
    "deny": [
      "Bash(rm *)",
      "Bash(git push --force)"
    ]
  }
}
```

- `allow` に書いたツールは確認なしで実行される
- `deny` に書いたツールは実行がブロックされる
- `Bash(パターン)` で特定のコマンドだけ許可/拒否できる

---

## 実務での設定例

### 開発中（ある程度自由に）

```json
{
  "permissions": {
    "allow": [
      "Read", "Glob", "Grep", "Write", "Edit",
      "Bash(npm *)",
      "Bash(git add *)",
      "Bash(git commit *)",
      "Bash(git status)",
      "Bash(git diff *)",
      "Bash(git log *)"
    ],
    "deny": [
      "Bash(git push --force *)",
      "Bash(rm -rf *)"
    ]
  }
}
```

### CI / 自動化（最小権限）

```json
{
  "permissions": {
    "allow": [
      "Read", "Glob", "Grep",
      "Bash(npm test)",
      "Bash(npm run lint)"
    ]
  }
}
```

書き込み系は一切許可しない。読み取りとテスト実行だけ。

## やっておくと安全なこと

### 1. git の状態を意識する

Claude Code は git 管理下のファイルを編集する。変更が気に入らなければ `git checkout` で戻せる。作業前にコミットしておくと安心。

### 2. 危険なコマンドは deny に入れておく

```json
{
  "permissions": {
    "deny": [
      "Bash(git push --force *)",
      "Bash(rm -rf *)",
      "Bash(docker rm *)",
      "Bash(DROP TABLE *)"
    ]
  }
}
```

### 3. 本番環境の認証情報を渡さない

Claude Code のセッション内で `export AWS_ACCESS_KEY_ID=...` のようなことはしない。`.env` に本番の認証情報を入れている場合、CLAUDE.md に「.env を読むな」と書いておく。

### 4. Plan モードで先に確認する

大きな変更をさせる前に Plan モードで実行計画を確認してから実際の作業に移ると事故が減る。

---

## まとめ

- Claude Code は全操作に権限チェックがある
- `.claude/settings.json` でツール単位の許可/拒否を設定できる
- 「自動で許可するもの」と「絶対にブロックするもの」を明示しておくと安全
- 本番の認証情報はセッションに渡さない
- 不安なときは Plan モードで先に計画を確認する
