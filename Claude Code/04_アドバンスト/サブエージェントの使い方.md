# サブエージェントの使い方

## こういうときに困る

「Claude に大きなタスクを頼んだが、途中でコンテキストが膨れ上がり、後半の判断がおかしくなった」「セキュリティチェックとコード実装を同じセッションでやらせたら、前者の結果が後者に干渉した」。

1 つのセッションで何でもやらせると、コンテキストが汚染されたり、集中力が散漫になったりする。大きな仕事ほど、専門家に分担させる仕組みが欲しい。

それを解決するのが **サブエージェント（Sub-agents）**。

---

## サブエージェントとは

メインの Claude セッションから、独立したコンテキストウィンドウを持つ「別の Claude インスタンス」を起動して作業を委譲できる機能。

サブエージェントは：
- **独立したコンテキストを持つ**（メインセッションの会話履歴を引き継がない）
- **専用の指示（system prompt）を与えられる**（「セキュリティレビュー専門」など）
- **使えるツールを制限できる**（読み取り専用にする等）
- **並列に複数起動できる**

---

## ビルトインサブエージェント

カスタム作成しなくても、以下のエージェントがデフォルトで使える：

| 名前 | モデル | 用途 |
|------|--------|------|
| Explore | Haiku | 読み取り専用のコードベース調査 |
| Plan | 親と同じ | Plan Mode での事前計画 |
| General-purpose | 親と同じ | 調査と変更が必要な複雑なタスク |

---

## カスタムサブエージェントの作り方

### 方法 1：`/agents` コマンドで作る

セッション内で `/agents` を実行 → 「Create new agent」を選択。

### 方法 2：Markdown ファイルを直接作る（推奨）

`.claude/agents/` ディレクトリに Markdown ファイルを置く。

```markdown
---
name: code-reviewer
description: コード品質をレビューする。コード変更後に積極的に使う。
tools: Read, Grep, Glob, Bash
model: sonnet
permissionMode: plan
---

あなたはシニアコードレビュアーです。品質・セキュリティ・保守性の観点でコードを確認してください。
指摘は箇条書きで。修正は行わない。
```

### ファイルの置き場所

| 場所 | スコープ |
|------|---------|
| `.claude/agents/` | 現プロジェクトのみ（チームで共有） |
| `~/.claude/agents/` | 全プロジェクト共通（個人設定） |

---

## フロントマターの主なフィールド

| フィールド | 説明 |
|-----------|------|
| `name` | エージェント名（必須） |
| `description` | Claude がいつ委譲するかの判断基準（必須） |
| `tools` | 使用を許可するツールのホワイトリスト |
| `disallowedTools` | 使用を禁止するツールのブラックリスト |
| `model` | `sonnet` / `haiku` / `opus` / `inherit` |
| `permissionMode` | `default` / `plan` / `acceptEdits` |
| `maxTurns` | 最大ターン数（暴走防止） |
| `isolation: worktree` | 独立した git worktree で実行 |

---

## 実際の使い方

### Claude に自動で判断させる

`description` に「コード変更後に積極的に使う」と書いておけば、Claude が適切なタイミングで自動委譲する。

### 明示的に呼び出す

```
> @"code-reviewer (agent)" この変更をレビューして
```

---

## コスト注意点

マルチエージェントワークフローは、単一エージェントと比べて **4〜7 倍のトークンを消費する**。

コストを抑えるには：
- サブエージェントのモデルは `haiku` を使う（調査系タスクは十分）
- `maxTurns` で上限を設ける
- 本当に分離が必要なタスクだけに使う

---

## サブエージェントが特に有効な場面

- **調査と実装を分けたいとき**：調査エージェントに情報を集めさせ、メインに渡す
- **コンテキストを汚染させたくないとき**：大量のログや中間出力を別エージェントに隔離する
- **同じ処理を複数対象に並列実行したいとき**：`isolation: worktree` で複数ファイルへの処理を同時に走らせる
