# フック（Hooks）の使い方

## フックとは

Claude Code が特定のアクション（ファイル保存、コマンド実行など）を行う前後に、自分で定義したシェルコマンドを自動実行する仕組み。lint の自動実行や通知の送信など、毎回手で指示しなくても勝手に走るようにできる。

---

## 設定場所

`.claude/settings.json`（プロジェクト単位）または `~/.claude/settings.json`（グローバル）に書く。

```json
{
  "hooks": {
    "イベント名": [
      {
        "matcher": "対象のパターン（省略可）",
        "command": "実行するシェルコマンド"
      }
    ]
  }
}
```

---

## 使えるイベント

| イベント | タイミング |
|---------|-----------|
| `PreToolUse` | ツール実行の **前** |
| `PostToolUse` | ツール実行の **後** |
| `Notification` | Claude が通知を出したとき |
| `Stop` | Claude がターン終了したとき |
| `SubagentStop` | サブエージェントがターン終了したとき |

---

## 実用例

### ファイル保存後に自動で lint

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": "npx eslint --fix $CLAUDE_FILE_PATH"
      }
    ]
  }
}
```

`Write` または `Edit` ツールが使われた後に ESLint が走る。

### Terraform ファイル編集後に自動フォーマット

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": "if echo \"$CLAUDE_FILE_PATH\" | grep -q '\\.tf$'; then terraform fmt -recursive \"$(dirname \"$CLAUDE_FILE_PATH\")\"; fi"
      }
    ]
  }
}
```

`.tf` ファイルが編集されたときだけ `terraform fmt -recursive` を走らせる。ただし **フォーマットだけなら VSCode の Terraform 拡張（HashiCorp Terraform）で `editor.formatOnSave` を有効にするほうが王道**。エディタ側でやるほうが設定も簡単で、Claude Code を使わない通常の開発でも効く。

フックが活きるのはフォーマット以外の処理を組み合わせたい場合。例えば `terraform validate` をセットで走らせる：

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": "if echo \"$CLAUDE_FILE_PATH\" | grep -q '\\.tf$'; then cd \"$(dirname \"$CLAUDE_FILE_PATH\")\" && terraform validate; fi"
      }
    ]
  }
}
```

`validate` が失敗しても PostToolUse なので実行自体はブロックされないが、エラー出力が Claude にフィードバックされるので、Claude が自分で修正しにいく。

### コミット前にテスト実行

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "if echo \"$CLAUDE_TOOL_INPUT\" | grep -q 'git commit'; then npm test; fi"
      }
    ]
  }
}
```

Claude が `git commit` を含むコマンドを実行しようとしたら、先にテストを走らせる。テストが失敗（exit code != 0）するとツール実行がブロックされる。

### 長時間タスク完了時に通知

```json
{
  "hooks": {
    "Stop": [
      {
        "command": "notify-send 'Claude Code' '作業が完了しました' 2>/dev/null; echo done"
      }
    ]
  }
}
```

## 使える環境変数

フックのコマンド内では以下の環境変数が使える。

| 変数 | 内容 |
|------|------|
| `$CLAUDE_TOOL_NAME` | 実行されたツール名（`Write`, `Bash` など） |
| `$CLAUDE_TOOL_INPUT` | ツールに渡された入力（JSON 文字列） |
| `$CLAUDE_FILE_PATH` | 対象ファイルのパス（ファイル操作時） |
| `$CLAUDE_SESSION_ID` | 現在のセッション ID |

---

## 注意点

- フックの exit code が 0 以外だと `PreToolUse` はツール実行をブロックする。意図しないブロックを避けるために、失敗してもいいコマンドには `|| true` をつけておく
- フックは同期実行される。重い処理を入れるとレスポンスが遅くなる
- セキュリティに注意。`$CLAUDE_TOOL_INPUT` をそのまま eval に渡すとインジェクションの危険がある

---

## まとめ

- フックは「ツール実行の前後に自動でシェルコマンドを走らせる」仕組み
- lint、テスト、通知など、毎回指示するのが面倒な処理に使う
- `.claude/settings.json` に書くだけで設定できる
- `PreToolUse` のフックが失敗するとツール実行がブロックされるので、意図的なガードとして使える
