# コストとプランの選び方

## 使い始める前に知っておきたいこと

Claude Code を使い始めると「思ったより高い」「プランが複数あってどれを選べばいいかわからない」という状況になりがち。事前に料金の仕組みを把握しておくと判断しやすい。

---

## 実際どれくらいかかるか

公式の実績値：
- 平均：**1 開発者あたり $6/日**
- 90% のユーザー：**$12/日以下**
- 月換算：**$100〜200/人**（ただし使い方による差が大きい）

重い作業（大規模リファクタリング・長いコードベースの調査）を繰り返すと上振れしやすい。

---

## プランの選び方

| プラン | 向いている人 | デフォルトモデル |
|--------|-------------|----------------|
| **Claude Pro** | 個人で試したい | Sonnet 4.6 |
| **Claude Max** | 個人でヘビーに使う | Opus 4.6 |
| **Team Standard** | 小〜中規模チーム | Sonnet 4.6 |
| **Team Premium** | Opus を使いたいチーム | Opus 4.6 |
| **Enterprise** | SSO・コンプライアンスが必要 | Opus 4.6 利用可 |
| **API（従量課金）** | 自動化・CI/CD 組み込み | 任意のモデルをフル指定 |

**迷ったら：**
- 個人で試す → Pro から始める
- 自動化・CI に使う → API（従量課金）
- チームで使う → Team Standard で様子を見る

---

## コストを把握するコマンド

```
/cost    # 現在のセッションのトークン使用量（API 課金ユーザー向け）
/stats   # 使用パターンの確認（サブスクリプションユーザー向け）
```

---

## コストを抑える実践的な方法

### 1. 不要に長いセッションを続けない

Claude Code は会話履歴をすべて毎回送信する。長くなるほどコストが上がる。タスクが終わったら `/clear` で履歴をリセットする。

### 2. 軽いタスクは Haiku を使う

```bash
claude -p "コミットメッセージを作って" --model claude-haiku-4-5
```

Haiku は Sonnet の数分の一のコストで動く。

### 3. サブエージェントには Haiku を割り当てる

調査・検索系のサブエージェントには Haiku を指定する（`model: haiku` をフロントマターに書く）。マルチエージェントはトークン消費が大きいので特に効果がある。

### 4. `--bare` フラグを CI で使う

CI/CD では不要な CLAUDE.md・hooks のロードを防ぐ `--bare` フラグを使う。余分なコンテキストが削減できる。

### 5. thinking は必要なときだけ

`/effort high` や `ultrathink` は thinking トークンを大量消費する。普段は `medium`（デフォルト）のまま使い、本当に詰まったときだけ使う。

---

## チームで使う場合のレート制限の目安

チームが大きいほど、1 人あたりの制限を下げてトークン枯渇を防ぐ必要がある。

| チーム規模 | 推奨 TPM/人 |
|-----------|------------|
| 1〜5 人 | 200k〜300k |
| 5〜20 人 | 100k〜150k |
| 20〜50 人 | 50k〜75k |
| 50〜100 人 | 25k〜35k |
