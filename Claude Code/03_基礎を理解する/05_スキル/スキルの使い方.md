# スキル（Skills）の使い方

## スキルとは

Claude Code に「こういうときはこう動いて」という指示セットを登録する仕組み。`/スキル名` で手動実行もできるし、会話の文脈に合えば Claude が自動で読み込んでくれる。

旧名称の「カスタムスラッシュコマンド」（`.claude/commands/`）と同じもの。スキルのほうが新しく、補助ファイルや細かい制御ができる。

---

## 置き場所

| 置き場所 | パス | 使える範囲 |
|---------|------|-----------|
| **個人用** | `~/.claude/skills/<スキル名>/SKILL.md` | 自分の全プロジェクト |
| **プロジェクト用** | `.claude/skills/<スキル名>/SKILL.md` | このプロジェクトだけ（git で共有可） |

---

## 最小構成

`.claude/skills/review/SKILL.md` を作る：

```yaml
---
name: review
description: PRのコードレビューを行う
---

以下の観点でコードレビューしてください：

1. ロジックの誤り（境界値、null考慮漏れ）
2. セキュリティ（認可チェック、入力バリデーション）
3. パフォーマンス（N+1クエリ、不要なループ）
4. エラーハンドリング（外部API失敗時の挙動）

指摘は [must] / [nit] / [question] のプレフィックスをつけて。
```

使い方：

```
/review src/api/users.ts
```

## よく使いそうなスキル例

### Issue 修正

```yaml
---
name: fix-issue
description: GitHub Issue を修正する
disable-model-invocation: true
---

GitHub Issue #$ARGUMENTS を修正してください。

1. Issue の内容を読む
2. 関連するコードを探す
3. 修正を実装する
4. テストを書く
5. コミットを作成する
```

```
/fix-issue 42
```

### コミット

```yaml
---
name: commit
description: 変更内容からコミットメッセージを作成してコミットする
disable-model-invocation: true
---

1. git diff と git status を確認
2. 変更内容を分析
3. Conventional Commits 形式でコミットメッセージを作成
4. git add して git commit する
```

### テスト生成

```yaml
---
name: add-tests
description: 指定ファイルのテストを生成する
---

$ARGUMENTS のテストを作成してください。

- 正常系、異常系、境界値をカバー
- 既存のテストファイルの書き方に合わせる
- モックは最小限に
```

## 重要な設定オプション

### Claude に自動で使わせたくない場合

デプロイやコミットなど、人間がタイミングを決めたいものには `disable-model-invocation: true` をつける。

```yaml
---
name: deploy
description: 本番デプロイ
disable-model-invocation: true
---
```

これがないと、Claude が「デプロイしたほうがよさそう」と判断したら勝手に実行する可能性がある。

### 引数を使う

`$ARGUMENTS` で `/スキル名` の後ろに渡した値を受け取れる。

```yaml
$ARGUMENTS[0]  # 第1引数
$ARGUMENTS[1]  # 第2引数
$0, $1         # 短縮形
```

### 動的にコマンド結果を埋め込む

`` !`コマンド` `` でシェルコマンドの実行結果をスキル内に展開できる。

```yaml
---
name: pr-review
description: 現在のPRをレビューする
---

## PR の差分
!`gh pr diff`

## 変更ファイル
!`gh pr diff --name-only`

上記の差分をレビューしてください。
```

Claude に渡る時点でコマンドの実行結果に置き換わっている。

## 補助ファイルを置く

スキルはディレクトリ単位なので、テンプレートやスクリプトを一緒に置ける。

```
.claude/skills/generate-api/
├── SKILL.md           # メインの指示
├── template.ts        # 生成時のテンプレート
└── examples/
    └── sample.ts      # 出力例
```

`SKILL.md` から参照すれば、Claude が必要に応じて読み込む。

---

## まとめ

- スキルは「繰り返す作業の指示書」を登録する仕組み
- `.claude/skills/<名前>/SKILL.md` を置くだけで使える
- `/スキル名` で手動実行、または Claude が自動で使う
- 勝手に実行されたくないものには `disable-model-invocation: true`
- チームで共有するなら git 管理すれば全員が同じスキルを使える
