# プロジェクト概要

技術的な学びやナレッジをトピック別にまとめた個人リポジトリ。コードを書くプロジェクトではなく、学習メモ・記事・スニペットの蓄積が目的。

## リポジトリ構造

- トップレベルのディレクトリがトピック（例: `Terraform/`, `Python/`, `Claude Code/`）
- 各トピック内はファイル名で分類（番号プレフィックスを付ける場合もあるが必須ではない）
- 番号付きサブディレクトリで章立てする場合もある（例: `Terraform/01_イントロダクション/`）

## 記事を書くときのルール

- 日本語で書く
- 無駄に長くしない。読む側の時間を尊重する
- 新しいトピックは既存のディレクトリ構造に合わせる

### 書き方のスタイル

**「現場でこういう問題が起きたとき、これで解決できる」という構成で書く。**

読み手が「あ、自分がいま困ってるのと同じ場面だ」と気づけることがゴール。公式ドキュメントの言葉の言い換えにならない。

- **冒頭は問題・場面から入る**: どういうときに困るのか、なぜこれが必要なのかを最初に書く
- **機能の説明はその後**: 構文・使い方は「問題を解決する手段」として位置づける
- **コード例は場面ベース**: 「こういうときにこう書く」という形にする。網羅的なサンプルより、実際に書く場面に近いものを選ぶ

**NG 例**（公式ドキュメントの言い換え）:
> 「data ブロックは既存リソースを参照する仕組みです」→ 構文の説明へ

**OK 例**（問題・場面ベース）:
> 「Terraform は自分が管理しているリソースしか知らない。でも手動で作った VPC や別チームが管理している Route53 など、管理外のリソースを参照したい場面がある。それを解決するのが data ブロック」

### セキュリティ記事での追加視点

セキュリティ系の記事を書くときは、以下の視点を意識する。

**「そのアクションのトリガーは何か？」を常に問う**

例：「Athena でログを調査したい」のトリガーは現実には侵害後の後追い調査。それを正直に書いた上で「だから侵害前にやっておくべきことは何か」に繋げる。ツールの説明ではなく、**いつ・なぜそれが必要になるか**から入る。

**提案者としての言語化を添える**

技術的な正しさだけでなく、ビジネス側に伝わる言い方も添える。

| 技術的な言い方 | ビジネス側に刺さる言い方 |
|---|---|
| CloudTrail を有効化する | 「誰が何をしたか」の記録がないと、インシデント時に原因調査も証拠収集もできない |
| GuardDuty を有効化する | 社内リソースへの不正アクセスを24時間監視する仕組みを月数万円で入れられる |

**用途が決まっていないツールを入れる提案はしない**

「早めに入れておけばログが溜まる」は根拠にならない。何に使うかが決まってから導入を提案する（シェルフウェア問題）。

## やらないこと

- コードの実行環境やビルドの設定（このリポジトリにはない）
- 英語で書く必要はない
- 過度な装飾やフォーマット統一の強制

## Confluence 自動同期

`Confluence/` ディレクトリの内容は `main` push 時に GitHub Actions で Confluence に自動同期される。

### ディレクトリ構造とページ階層

git のディレクトリ構造 = Confluence のページ階層。

```
Confluence/
├── Confluence.md              ← Confluence セクションの親ページ
├── 運用手順.md                ← Confluence 直下の子ページ
├── カテゴリ名/
│   ├── カテゴリ名.md          ← カテゴリの親ページ（フォルダ名と同名にすること）
│   └── 記事.md
└── ...
```

### front matter（必須）

```markdown
<!-- Space: harukaaibarapublic -->
<!-- Parent: 親ページのタイトル -->
<!-- Title: このページのタイトル -->
```

### 注意事項

- タイトル変更・ファイル移動・ファイル削除をした場合、**Confluence 上の旧ページは自動削除されない**。手動で削除すること
- 新しいカテゴリを追加するときは、**フォルダ名と同名の `.md` ファイル**（インデックスページ）を必ず作ること
- 詳細な操作手順は `Confluence/運用手順.md` を参照
