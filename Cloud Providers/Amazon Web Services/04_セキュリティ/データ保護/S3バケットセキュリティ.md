# S3 バケットセキュリティ

「S3 バケットが公開状態になっていて個人情報が漏洩した」——S3 に関するセキュリティインシデントは後を絶たない。デフォルト設定のまま使い続けるのが最大のリスク。

---

## パブリックアクセスブロック（最優先で設定）

新規バケットはデフォルトでパブリックアクセスがブロックされているが、既存バケットや Terraform で作ったバケットは明示的に設定する。

```bash
# アカウント全体でパブリックアクセスをブロック（最も確実）
aws s3control put-public-access-block \
  --account-id 123456789012 \
  --public-access-block-configuration \
    BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true

# 個別バケットにも設定
aws s3api put-public-access-block \
  --bucket my-bucket \
  --public-access-block-configuration \
    BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true
```

| 設定項目 | 内容 |
|---|---|
| BlockPublicAcls | パブリック ACL の設定を拒否 |
| IgnorePublicAcls | 既存のパブリック ACL を無視 |
| BlockPublicPolicy | パブリックアクセスを許可するポリシーを拒否 |
| RestrictPublicBuckets | パブリックポリシーがあってもアクセスを制限 |

## サーバーサイド暗号化

```bash
# SSE-S3（AWS 管理キー）
aws s3api put-bucket-encryption \
  --bucket my-bucket \
  --server-side-encryption-configuration '{
    "Rules": [{
      "ApplyServerSideEncryptionByDefault": {
        "SSEAlgorithm": "AES256"
      },
      "BucketKeyEnabled": true
    }]
  }'

# SSE-KMS（カスタマーマネージドキー）
aws s3api put-bucket-encryption \
  --bucket my-bucket \
  --server-side-encryption-configuration '{
    "Rules": [{
      "ApplyServerSideEncryptionByDefault": {
        "SSEAlgorithm": "aws:kms",
        "KMSMasterKeyID": "arn:aws:kms:ap-northeast-1:123456789012:key/key-id"
      },
      "BucketKeyEnabled": true
    }]
  }'
```

`BucketKeyEnabled: true` にすると KMS API コールが減ってコストを抑えられる。

## バケットポリシーで HTTPS 通信を強制

平文 HTTP でのアクセスを拒否する。

```json
{
  "Statement": [{
    "Sid": "DenyNonSSL",
    "Effect": "Deny",
    "Principal": "*",
    "Action": "s3:*",
    "Resource": [
      "arn:aws:s3:::my-bucket",
      "arn:aws:s3:::my-bucket/*"
    ],
    "Condition": {
      "Bool": {
        "aws:SecureTransport": "false"
      }
    }
  }]
}
```

## アクセスログの有効化

誰がいつ何にアクセスしたかを記録する。インシデント発生時の調査に必須。

```bash
aws s3api put-bucket-logging \
  --bucket my-bucket \
  --bucket-logging-status '{
    "LoggingEnabled": {
      "TargetBucket": "my-access-logs-bucket",
      "TargetPrefix": "s3-access-logs/my-bucket/"
    }
  }'
```

ログバケットは別の S3 バケットに分けておく（アプリバケットと同一にしない）。

## Amazon Macie で機密データを検出

S3 に機密データ（クレジットカード番号・個人情報・パスワード）が入っていないかを自動スキャンする。

```bash
# Macie を有効化
aws macie2 enable-macie

# S3 バケットの分類ジョブを作成
aws macie2 create-classification-job \
  --name "scan-prod-buckets" \
  --job-type ONE_TIME \
  --s3-job-definition '{
    "bucketDefinitions": [{
      "accountId": "123456789012",
      "buckets": ["my-bucket"]
    }]
  }'
```

## Config ルールで設定ドリフトを継続監視

| ルール名 | チェック内容 |
|---|---|
| s3-bucket-public-read-prohibited | パブリック読み取りアクセスが無効か |
| s3-bucket-public-write-prohibited | パブリック書き込みアクセスが無効か |
| s3-bucket-ssl-requests-only | HTTPS のみの設定か |
| s3-bucket-server-side-encryption-enabled | 暗号化が有効か |
| s3-bucket-logging-enabled | アクセスログが有効か |

AWS Security Hub の AWS Foundational Security Best Practices を有効にすると、これらのチェックが自動的に実行される。
