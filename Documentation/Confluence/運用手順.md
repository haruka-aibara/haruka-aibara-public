# 運用手順

GitHub の Markdown を Confluence に自動同期する仕組みの日常操作まとめ。

---

## 仕組みのおさらい

```
Confluence/
├── Confluence.md          ← 親ページ（Confluence セクションのトップ）
├── 運用手順.md            ← このページ
├── 同期方法/
│   ├── 同期方法.md        ← サブカテゴリの親ページ（フォルダ名と同名）
│   └── *.md               ← 記事
└── セットアップ/
    ├── セットアップ.md
    └── *.md
```

- `main` ブランチに push すると GitHub Actions が自動実行される
- ワークフローはファイルを深さ順・インデックスページ優先で処理するため、親ページが先に作成される
- **Confluence → GitHub の同期はない**。Confluence 側で直接編集しても上書きされるので、必ず GitHub で編集すること

---

## front matter テンプレート

すべての `.md` ファイルの先頭に必須。

```markdown
<!-- Space: harukaaibarapublic -->
<!-- Parent: 親ページのタイトル -->
<!-- Title: このページのタイトル -->
```

| 項目 | 値 |
|---|---|
| `Space` | 常に `harukaaibarapublic` |
| `Parent` | Confluence 上の親ページタイトルをそのまま書く |
| `Title` | Confluence 上のページタイトル |

---

## 記事の内容を更新する

1. 対象の `.md` ファイルを編集する
2. `git commit & push` → ワークフローが自動実行される

```bash
git add Confluence/同期方法/GitHubMarkdownをConfluenceに自動同期する方法.md
git commit -m "GitHubMarkdownをConfluenceに自動同期する方法: xxx を追記"
git push origin main
```

---

## 記事のタイトルを変更する

> **注意**: タイトルを変えると mark は旧ページを見つけられず新規作成する。**旧ページが Confluence 上に残るため手動削除が必要。**

1. `.md` ファイルの `<!-- Title: -->` を変更する
2. `git commit & push`
3. Confluence にログインし、旧タイトルのページを手動削除する

---

## 新しい記事を追加する

既存のカテゴリ（`同期方法/` や `セットアップ/`）に追加する場合。

1. 対象フォルダに `.md` ファイルを作成する
2. front matter を記述する（`Parent` はフォルダの親ページタイトルと合わせる）
3. `git commit & push`

```markdown
<!-- Space: harukaaibarapublic -->
<!-- Parent: セットアップ -->
<!-- Title: 新しい記事のタイトル -->

# 新しい記事のタイトル

本文...
```

---

## 新しいカテゴリ（サブフォルダ）を追加する

1. `Confluence/` 下にフォルダを作成する
2. **フォルダ名と同名の `.md` ファイル**を作成する（これがカテゴリの親ページになる）
3. 記事ファイルを追加する
4. `git commit & push`

```
Confluence/
└── 新カテゴリ/
    ├── 新カテゴリ.md      ← Parent: Confluence
    └── 記事.md            ← Parent: 新カテゴリ
```

インデックスページの front matter:

```markdown
<!-- Space: harukaaibarapublic -->
<!-- Parent: Confluence -->
<!-- Title: 新カテゴリ -->

カテゴリの説明文。
```

---

## 記事を別のカテゴリに移動する

> **注意**: ファイルを移動すると mark は旧ページを見つけられず新規作成する。**旧ページが Confluence 上に残るため手動削除が必要。**

1. ファイルを移動する（`git mv`）
2. front matter の `<!-- Parent: -->` を移動先カテゴリに合わせて変更する
3. `git commit & push`
4. Confluence にログインし、旧ページを手動削除する

```bash
git mv Confluence/同期方法/記事.md Confluence/セットアップ/記事.md
# front matter の Parent を「セットアップ」に変更
git add -A
git commit -m "記事を同期方法 → セットアップに移動"
git push origin main
```

---

## 記事を削除する

> mark は削除を自動検知しない。**Confluence 上のページは手動削除が必要。**

1. `.md` ファイルを削除する
2. `git commit & push`
3. Confluence にログインし、対象ページを手動削除する

```bash
git rm Confluence/同期方法/不要な記事.md
git commit -m "不要な記事を削除"
git push origin main
```

---

## ワークフローを手動実行する

GitHub Actions タブ → **Sync to Confluence** → **Run workflow** → **Run workflow**

Confluence 上のページ構造が壊れた場合は、先に Confluence 側の対象ページを削除してから実行する。
