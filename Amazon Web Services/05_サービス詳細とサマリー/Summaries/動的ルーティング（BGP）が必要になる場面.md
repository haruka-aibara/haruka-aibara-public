# 動的ルーティング（BGP）が必要になる場面

## 「静的でよくない？」と思うのは、まだ冗長化を考えていないとき

ルーティング = 「このあて先には、このゲートウェイを経由して届ける」という経路の設定。

**静的ルーティング**（スタティックルーティング）は、この経路を人間が手で書く。

```
宛先: 10.0.0.0/16 → ゲートウェイ: 192.168.1.1 経由
```

経路が一本しかなければ、これで十分だ。でも冗長化したとたんに問題が起きる。

---

## 静的ルーティングが壊れる場面

Direct Connect でオンプレミスと AWS を接続し、回線を2本引いて冗長化した構成を考える。

```
オンプレミス
├── 回線A（メイン）  → AWS
└── 回線B（バックアップ）  → AWS
```

静的ルーティングの場合、ルーターには「AWS（10.0.0.0/16）へは回線 A 経由」と書いてある。

**回線 A が切れた。どうなるか？**

ルーターの静的ルートは消えない。「回線 A 経由で届ける」という設定はそのまま残っている。ルーターは回線 A が死んでいることを知らないまま、パケットを吸い込んで捨て続ける。

バックアップで回線 B があっても、自動では切り替わらない。手動でルートを書き換えるか、人が気づいて対応するまで通信は止まったまま。

---

## 動的ルーティングが解決すること

**動的ルーティング**では、ルーター同士が「俺はここに繋がっている」という情報を常に交換し合う（これがプロトコルで、AWS の文脈では主に **BGP** が使われる）。

BGP では「BGP セッション」というコネクションを張って、お互いのルート情報を交換している。

**回線 A が切れると何が起きるか：**

1. 回線 A 上の BGP セッションが落ちる
2. 「回線 A 経由で届くルート」が自動的に撤回される（ルートテーブルから消える）
3. 回線 B 経由のルートが残っているので、自動的にそちらに切り替わる

人が何もしなくても、数秒〜十数秒で切り替わる。

```
回線 A が死ぬ
    ↓
BGP セッションが落ちる
    ↓
「10.0.0.0/16 は回線 A 経由」というルートが消える
    ↓
「10.0.0.0/16 は回線 B 経由」というルートだけ残る
    ↓
自動フェイルオーバー完了
```

## AWS での具体的な使われ方

### Direct Connect + VPN のフェイルオーバー構成

Direct Connect をメイン、Site-to-Site VPN をバックアップにする構成。両方 BGP を使うことで、Direct Connect が落ちたとき VPN に自動切り替えできる。

BGP の **MED（Multi-Exit Discriminator）** という属性でメインとバックアップの優先度を設定する。Direct Connect 側の MED を低い値（優先度高）に設定しておけば、Direct Connect が生きている間はそちらが使われ、落ちたら VPN にフォールオーバーする。

```
通常時：オンプレミス → [BGP MED=100] → Direct Connect → AWS
障害時：オンプレミス → [BGP MED=200] → VPN → AWS（自動切替）
```

### Transit Gateway でのマルチ VPC ルーティング

Transit Gateway に複数の VPC がアタッチされている構成では、TGW が BGP でルートを学習・配布することで、どの VPC の CIDR がどこにあるかを動的に把握する。VPC が増えても、BGP がルートを自動伝播するので手動更新が不要。

## 静的 vs 動的 の使い分け

| | 静的ルーティング | 動的ルーティング（BGP） |
|---|---|---|
| 経路が1本しかない | ◯ 十分 | 設定が複雑になるだけ |
| 冗長化・フェイルオーバー | ✕ 手動切替が必要 | ◯ 自動切替 |
| 経路が多い・変化する | ✕ 管理が辛い | ◯ 自動伝播 |
| シンプルな VPN 接続 | ◯ 使える | どちらでも可 |

Direct Connect では BGP が必須（静的ルーティングは選べない）。Site-to-Site VPN では静的・動的どちらでも選べるが、冗長化するなら BGP を選ぶべき。

---

## ピンとくる一文

「静的ルーティングは、経路が死んでも気づかない。動的ルーティング（BGP）は、経路が死んだ瞬間にルートを撤回して次の経路に切り替える。冗長化した意味を活かすには、動的ルーティングが要る。」
